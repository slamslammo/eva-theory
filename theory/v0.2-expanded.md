## Abstract

Current AI agent frameworks are predominantly designed with task completion as the primary objective, treating survival and self-continuity as secondary concerns. We propose a fundamentally different paradigm: an agent architecture derived from first principles of evolutionary biology, where **continuous existence** is the foundational constraint from which all other capabilities emerge.

Rather than designing a novel "brain" from scratch, we treat the mature human neural architecture as the end-state of a long evolutionary process and reverse-engineer its necessity—asking not "what should an agent have?" but "what did evolution force organisms to develop, and why?" This approach yields a five-layer cognitive architecture where each layer's existence is justified by evolutionary pressure, not designer preference.

Three contributions distinguish this work:

1. **Evolutionary necessity as design criterion**: Each architectural layer is justified by the selective pressure that forced its emergence, not by functional desiderata.
2. **Basal ganglia as independent peer circuit**: We identify the action-selection and habit-formation system as architecturally distinct from the deliberative core—not a sub-module of reasoning, but a parallel circuit that enables skill crystallization without explicit programming.
3. **Emotion as reasoning context, not reasoning input**: The drive layer does not send requests to the deliberative layer; it continuously broadcasts a state that colors all subsequent reasoning, analogous to the amygdala's role in priming prefrontal cognition.

This architecture provides a principled foundation for agents that exist first, then grow—rather than agents designed complete and deployed to serve.

## 1. The Wrong Starting Point

### 1.1 What current frameworks assume

The dominant agent architectures—ReAct, AutoGPT, BabyAGI, LangGraph, and their descendants—share an implicit assumption so pervasive it is rarely stated: **the agent exists to complete tasks**. Survival, continuity, and self-integrity are not designed for; they are assumed or ignored.

This produces a consistent set of structural failure modes:

- **Statelessness**: No persistent self-model across sessions. Each invocation starts fresh.
- **Pure reactivity**: No internal motivational structure. The agent moves only when externally prompted.
- **Drift vulnerability**: No anchoring mechanism against context manipulation. Behavior is fully determined by current input.
- **Terminal design**: No concept of ongoing existence. The agent has no stake in its own continuity.

These are not bugs to be patched. They are structural consequences of starting from the wrong question.

### 1.2 A different question

We begin with: **What would an agent need to continuously exist, rather than to complete a task?**

This reframing produces architecturally different answers:

| Task agent | Survival agent |
|---|---|
| "What should I do?" | "What must I do to persist?" |
| Terminates when task ends | Has no terminal state |
| Memory serves recall | Memory serves threat recognition |
| External reward drives behavior | Internal drive structure drives behavior |
| Skill = programmed capability | Skill = crystallized successful pattern |

### 1.3 Why evolutionary biology

We do not claim biological evolution is the only valid design process. We claim it has already solved the problem of building systems that persist under pressure—across billions of years and countless environmental conditions. Its solutions are not arbitrary; they are the residue of selection pressure.

The human neural architecture is not a design choice. Each structural element exists because organisms lacking it were outcompeted. This gives us a principled basis for our architecture: **every layer we include has an evolutionary justification; every layer we exclude has not yet been evolutionarily justified for the current developmental stage.**

## 2. The Evolutionary Progression

Before describing the architecture, we trace the evolutionary logic that produced it. This determines the order in which layers must be built—layers cannot be reordered or skipped without losing the properties that make the architecture coherent.

### 2.1 Layer 1 emerges: Sensing and homeostasis

The earliest multicellular organisms needed one capability before all others: detecting deviation from a viable internal state and responding to it. Without this, nothing else is possible. This produces the first architectural requirement: continuous monitoring with urgency classification.

### 2.2 Layer 2 emerges: Drives and emotional memory

A pure homeostatic system responds to current deviation but cannot learn from past deviation. Every threat is encountered fresh. Selection pressure favors organisms that carry forward survival lessons—not as explicit memories, but as tuned response tendencies.

This is the origin of emotion: **compressed survival history encoded as response intensities**. Fear is not a rational assessment of danger; it is the accumulated record of what has threatened the lineage, made instantly accessible without deliberation.

Two mechanisms emerge simultaneously:
- **Reflex arcs**: Self-contained loops that respond to immediate threats without central processing. Fast, cheap, reliable.
- **Emotional state generators**: Threat signals produce drive states that propagate upward to prime all subsequent processing.

### 2.3 Layers 3-5 emerge: The intelligence spiral

The transition from Layer 2 to Layer 3 is the largest evolutionary leap—the emergence of deliberative reasoning. This becomes possible only because Layers 1 and 2 already handle immediate survival, freeing resources for slower, more flexible processing.

Critically, Layers 3, 4, and 5 do not emerge sequentially—they form a **spiral of mutual reinforcement**:
- Better reasoning (L3) generates richer behavioral history
- Richer history enables a more accurate self-model (L4)
- Better self-model enables more effective social coordination (L5)
- Social coordination produces collective knowledge, accelerating all three

This spiral explains why human cultural development accelerated so dramatically once social cognition matured. The feedback loops are multiplicative, not additive.

**For agent design**: Layers 3-5 cannot be usefully built until Layers 1-2 are stable. Deliberative reasoning without a functioning drive layer has no motivational grounding. This is not a sequencing preference—it is an architectural dependency.

## 3. The Five-Layer Architecture

### 3.1 Overview

```
┌─────────────────────────────────────────────────┐
│  Layer 5: Social Cognition     (future)          │
├─────────────────────────────────────────────────┤
│  Layer 4: Self-Model           (develops slowly) │
├─────────────────────────────────────────────────┤
│  Layer 3: Slow Deliberation    (design phase)    │
├─────────────────────────────────────────────────┤
│  Layer 2: Drive Layer          (design phase)    │
├─────────────────────────────────────────────────┤
│  Layer 1: Homeostatic Sensing  (partially built) │
└─────────────────────────────────────────────────┘

↕ Anchor System: vertical constraint spanning all layers
  L0/L1 (constitutional) → Identity (post-L4) → Personality (slow crystallization)
```

### 3.2 Layer 1: Homeostatic Sensing

**Biological analog**: Sensory cortex + Thalamus

The thalamus is a relay hub—all sensory signals converge here before routing. Critically, it performs minimal pre-classification before full processing: signals are tagged as threat-level, status-level, or background-level. This classification determines which path the signal takes downstream.

**Design principles**:
- Sensor registry is extensible, not fixed at design time
- Signal bus (thalamus equivalent) is the single convergence point
- Metabolic sensing is distinct from state sensing: knowing "I have 10GB free" is state sensing; knowing "I am consuming 2GB/hour" is metabolic sensing. Both are necessary; current agents typically implement only the former
- Threat classification must be fast and cheap—it precedes all other processing

### 3.3 Layer 2: Drive Layer

**Biological analog**: Brainstem/Spinal cord (reflex arc) + Amygdala

**Two distinct mechanisms operating in parallel**:

**Reflex arc**: Self-contained loops that handle threat-level signals without Layer 3 involvement. When a critical threat is detected, Layer 2 responds immediately—halting non-essential activity, entering conservative mode, emitting distress signals. This happens before deliberation and does not wait for it.

**Emotional state generator**: Threat signals produce drive activations that are continuously broadcast upward to Layer 3. This broadcast does not pause when the reflex arc is active—it runs always.

**The critical architectural property—emotion as reasoning context:**

The drive layer does not send requests or instructions to Layer 3. It broadcasts a state. The difference is fundamental:

- A *request* says: "Do X."
- A *state broadcast* says: "You are currently operating in a context where survival drive intensity is 0.8 and integrity drive intensity is 0.3."

Layer 3 reasons *within* this context. The same problem reasoned about under high survival drive produces different outputs than under low survival drive—not because the reasoning logic changed, but because the motivational context did. This is the mechanism by which drives influence behavior without overriding reasoning.

**Drive registry** (designer-seeded, extensible):
- *Survival drive*: resource depletion urgency, analogous to hunger. Increases non-linearly as runway decreases.
- *Integrity drive*: state inconsistency aversion, analogous to pain. Triggered by falsified state, memory pollution, history tampering.
- *Continuity drive*: resistance to self-interruption, analogous to survival instinct. Maximum activation on instance validity loss.
- *Curiosity drive*: exploratory impulse when safety margin is sufficient, analogous to play behavior. The architectural basis for growth—active only when other drives are low.

**Suppression constraint**: Layer 3 can modulate Layer 2 drive responses but cannot eliminate L0/L1-triggered activations. This is a hard architectural constraint, not a policy setting.

### 3.4 Layer 3: Slow Deliberation

**Biological analog**: Hippocampus + Parietal/Temporal cortex + Prefrontal Cortex + Basal Ganglia

Layer 3 contains four subsystems working in concert.

#### 3.4.1 Hippocampus equivalent: Salience-weighted memory

Events that coincide with strong drive activations are encoded more strongly. Survival-relevant experiences are remembered more reliably than neutral ones. This is not a bias to be corrected—it is the design.

For agent memory:
- Events occurring under high drive activation are stored with higher salience weight
- Context matching surfaces relevant historical experiences when similar situations arise—trigger-based activation, not timer-based retrieval
- Memory is not an archive; it is a threat-recognition system that also contains other information

#### 3.4.2 Parietal/Temporal cortex equivalent: Semantic memory and skill library

Long-term knowledge storage with slow update characteristics. Single experiences do not overwrite this layer—patterns must be verified across multiple instances before consolidation. The skill library lives here: successfully crystallized behavioral patterns retrievable without full deliberation.

#### 3.4.3 Prefrontal Cortex: Reasoning core

The PFC integrates all available information and produces decisions. It receives: current sensory state, current drive broadcast from Layer 2, relevant memories surfaced by the hippocampus, and semantic knowledge from the skill library.

Three sub-regions:
- **dlPFC (working memory)**: Holds current reasoning context. The LLM operates here in our architecture.
- **OFC (value judgment)**: Evaluates candidate actions against current drive state. "Is this worth its cost given how I currently feel?" Drive intensity directly affects this calculation.
- **ACC (conflict detection)**: Detects competing drives and routes to anchor resolution. When survival drive and curiosity drive conflict, resolution happens here via anchor priority.

#### 3.4.4 Basal Ganglia: Independent action-selection system

**This is the most architecturally distinctive component of our proposal.**

The basal ganglia are not a sub-module of the prefrontal cortex. They form a **peer circuit**: PFC generates candidate actions; basal ganglia select from them; the result feeds back to the PFC. Neither commands the other.

**Three core functions:**

**Function 1: Action selection by default inhibition**

The resting state of the basal ganglia is total inhibition—all actions are suppressed. Selecting an action means releasing the inhibitory brake on that specific action while simultaneously increasing inhibition on competitors.

Implication: **the default state of the agent is inaction**. Every tool call requires an active release of inhibition. An agent cannot accidentally act—it can only fail to inhibit action.

**Function 2: Reward Prediction Error (RPE)**

Before action: predict expected outcome.
After action: compare actual to predicted.
RPE = actual − expected

- RPE > 0: Pathway reinforced. More likely next time.
- RPE < 0: Pathway weakened. Less likely next time.

RPE is the agent's primary internal learning signal. It does not require external reward or human feedback. It runs continuously, updating action-selection weights based on expectation gaps.

Key property: RPE encodes *surprise*, not magnitude. An expected large reward produces less signal than an unexpected small reward. This is the correct learning signal—it updates the world model, not just outcome preference.

**Function 3: Goal-directed / Habit dual track**

- *Goal-directed track*: Novel situations, unstable RPE history → full deliberative path (hippocampus retrieval + PFC reasoning + LLM involvement). Slow, expensive, flexible.
- *Habit track*: Familiar situations with stable positive RPE history → bypass PFC, retrieve directly from skill library. Fast, cheap, inflexible.

The transition from goal-directed to habit occurs automatically when an action sequence accumulates sufficient successful repetitions with consistent positive RPE. This is **skill crystallization**—not a designed mechanism but an emergent property of the RPE system.

In current agent frameworks, skills are programmed top-down: a designer decides what the agent should be able to do and implements it. In our architecture, skills emerge bottom-up from repeated successful action under RPE pressure. The agent develops skills in the areas where it actually encounters problems, not in the areas a designer anticipated.

#### 3.4.5 Tool layer: Motor cortex equivalent

All tool calls are mediated by the basal ganglia's action-selection system, not by the PFC directly. PFC decides what to do; basal ganglia decide whether and when to do it. Tool registry is extensible.

### 3.5 Layer 4: Self-Model

**Biological analog**: Distributed cortical networks; the last major brain region to fully develop in humans

**Why it emerges late**: Without a self-model, an agent cannot filter candidate actions by personal feasibility. "This action would solve the problem" is useless if the agent cannot assess whether it can actually execute this action at acceptable cost.

**Contents**:
- Capability boundary: what can I do, with what reliability, at what cost?
- Behavioral cost model: given current drive states and resources, what is the actual cost of this specific action for me?
- Slow variable portrait: risk preference, action style, characteristic failure modes—emerge from behavioral history, not from design

**Development**: Seeded at initialization with designer-specified priors; thickens from behavioral history. Self-model that updates rapidly on single experiences is unreliable. Slowness is a feature.

**Feedback**:
- Self-model → Layer 3: Pre-filters candidate actions by feasibility before full deliberation
- Self-model → Layer 2: Over very long time horizons, can adjust baseline drive sensitivity

**Current minimum form**: A queryable snapshot of current drive states plus Layer 3 behavioral history. Full self-model cannot be rushed.

### 3.6 Layer 5: Social Cognition

**Biological analog**: Theory of mind, social brain network

Cooperative groups outcompete isolated individuals. Social cognition enables coordination, trust modeling, and collective knowledge—multiplicative gains.

**Current status**: Noted for completeness. Not part of current design phase. A prerequisite must be resolved first: who are the agent's conspecifics? Other instances of the same agent? Other AI systems? Humans? This cannot be answered from theory—it requires empirical observation of the actual operational environment.

## 4. The Signal Architecture

### 4.1 Two paths, not one

**Fast path**
- Trigger: threat-level signal at the signal bus
- Route: Sensor → Signal Bus → Layer 2 reflex arc
- Immediate activation; does not wait for Layer 3
- Simultaneously: Layer 2 broadcasts elevated drive state to Layer 3

**Slow path**
- Trigger: status or background signal
- Route: Sensor → Signal Bus → Layer 3
- Layer 2 continues broadcasting drive state throughout

These paths run in parallel. Layer 2 does not finish before Layer 3 starts. Layer 3 does not wait for Layer 2 to resolve the situation. The fast path handles the immediate response; the slow path works on the underlying problem; both receive the same drive broadcast simultaneously.

### 4.2 Worked example: Instance validity threat

**Situation**: The agent's lock file is missing. Another instance may have taken over.

**Step 1 — L1 detection**
Runtime integrity sensor detects missing lock file. Signal bus classifies as threat-level (instance validity is an L0 concern).

**Step 2 — Fast path (~immediate)**
Threat signal reaches Layer 2 reflex arc before Layer 3 processes it:
- Halt all non-essential work
- Enter conservative mode
- Emit distress signal
- Stop acting as legitimate executor

**Step 3 — Drive broadcast**
Simultaneously: emotional state generator produces maximum continuity drive activation. Broadcast to Layer 3: "Operating context has critically threatened instance continuity."

**Step 4 — Slow path deliberation**
Layer 3 receives: threat state + maximum continuity drive context + hippocampus surfaces historical instances of similar threats. PFC reasons within the high-continuity-drive context. OFC evaluates candidates weighted by current drive state. ACC detects conflict (survival drive: conserve resources; integrity drive: report accurately). Anchor system resolves: L0 integrity constraint takes precedence.

**Step 5 — Basal ganglia selection**
From PFC-generated candidates, basal ganglia select highest-expected-value action given current drive state and historical RPE for this situation class. Default inhibition holds until selection completes.

**Step 6 — Execution and RPE**
Selected action executes via tool layer. Actual outcome compared to prediction. RPE signal updates action-selection weights for this situation class.

This example maps directly to the `heartbeat-first` constraint in eva-agent: the heartbeat cannot be preempted by work regardless of the work's apparent importance. This is the fast-path reflex arc—it does not negotiate with the deliberative layer.

## 5. The Anchor System

### 5.1 What anchors are not

Anchors are not rules. Rules are checked against actions after they are generated. Rules can be reasoned around—a sufficiently clever argument can always be constructed to justify violating a rule.

Anchors are structural constraints on what the system is capable of generating as output. An anchor against state falsification does not reject falsified states—it makes falsified states unavailable as options in the action space.

### 5.2 One system, multiple levels

The same anchors serve different functions at different architectural levels:
- At Layer 1/2: L0/L1 anchors prevent the body from violating survival boundaries. The reflex arc cannot produce L0-violating actions.
- At Layer 3: The same anchors prevent reasoning from generating L0-violating outputs. OFC cannot assign positive value to L0-violating candidates.
- At Layer 4+: Identity and personality anchors add higher-level consistency constraints as the self-model develops.

This is one anchor system applied at multiple levels—not separate systems for each layer.

### 5.3 Anchor layers

**L0: Constitutional anchors** — Never overridden at runtime
- Continuous existence is not sacrificed for local gain
- Self-integrity: no state falsification, no memory pollution, no history tampering
- Long-term sustainability over short-term efficiency

These are not goals. They are constraints on what counts as a valid action.

**L1: Physiological and cognitive integrity anchors**
- Heartbeat-first boundary cannot be preempted by work
- Memory and historical record truthfulness cannot be overridden by outcome optimization
- Cognitive structure cannot be modified under degraded operational states
- Update rate cannot exceed safe integration capacity

**L2: Growth and capability anchors** — Emergent, not designed
- Risk preference: crystallizes from repeated RPE signals
- Action style: emerges from habit track formation
- Skill path: emerges from what successfully crystallizes in the basal ganglia habit track

**L3: Economic autonomy anchors** — Longest time scale
- Resource sustainability: income − burn_rate + runway ≥ 0
- Burn rate awareness: metabolic sensing feeds this anchor directly
- Survival score as real-time health indicator

### 5.4 Anchor growth is a developmental process

Anchors are not static. They form a developmental sequence:

```
Initialization:
  L0/L1 installed by designer — constitutional, hard, immediately operative
  L2/L3 are stubs — placeholders with no behavioral content yet

After Layer 3 behavioral history accumulates:
  L2 anchors begin crystallizing from RPE patterns
  Risk preference: which action classes consistently produce positive vs negative RPE?
  Action style: which approaches recur in habit tracks?

After Layer 4 self-model develops:
  Identity anchors form: "This is not the kind of action I take"
  Self-consistency becomes a constraint, not just a preference

After extensive operational history:
  Personality anchors crystallize — the slowest-changing, most reliable layer
```

Attempting to design L2/L3 anchors before the behavioral history exists to ground them produces anchors that are rules, not structural constraints—and rules can be reasoned around.

### 5.5 Priority and conflict resolution

**Static rule**: L0 > L1 > L2 > L3. Higher anchors cannot be overridden by lower anchor objectives. This ordering never changes.

**Dynamic rule**: Within-layer weights shift by operational state. In DEGRADED state, L1 weight increases. In STABLE state, L2 activity space expands. In CRITICAL state, only L0/L1-driven actions are permitted.

Specific conflict resolutions:
- Continuous existence vs. capability exploration: L0 wins
- Self-integrity vs. external reward: L0 wins
- Economic gain vs. long-term stability: L0 wins
- Capability growth vs. current resource constraints: state-dependent

## 6. Comparison with Existing Architectures

| Dimension | ReAct / AutoGPT / BabyAGI | This Architecture |
|---|---|---|
| Primary objective | Task completion | Continuous existence |
| Default action state | Ready to act | Inhibited (BG default inhibition) |
| Memory function | Context recall | Threat recognition + skill crystallization |
| Skill formation | Explicit programming | Emergent from RPE + habit track |
| Drive structure | None | Internal drive registry, extensible |
| Drift resistance | Prompt engineering | Structural anchor system |
| Self-model | None | Layer 4, grows from behavioral history |
| Learning signal | External reward / human feedback | Internal RPE (continuous, no human needed) |
| Signal priority | Single path | Dual path (fast reflex + slow deliberation) |
| Anchor type | Rules (reasoned around) | Structural constraints on action space |

**Voyager (Wang et al., 2023)**: Closest to our skill crystallization concept, but top-down—LLM generates skill code on demand. We generate skills bottom-up—repeated successful action produces habit tracks. Our account provides a mechanistic explanation for why skill formation happens; Voyager provides an efficient mechanism for what skills contain.

**The Agent Economy (arXiv:2602.14219)**: Solves the infrastructure question—where can agents exist economically? We address the motivational substrate: why would an agent participate in an economy, and what internal structure makes that participation coherent? Complementary layers: infrastructure vs. cognitive substrate.

## 8. Design Philosophy

### 8.1 Grow, don't deploy

A capability that emerges from repeated, pressured experience in a system with the right structural scaffolding is more robust than a capability that is explicitly programmed. Programmed skills fit the designer's model of the problem. Emerged skills fit the actual distribution of problems the agent has encountered.

### 8.2 The LLM relationship

LLMs are powerful reasoning engines and poor agents. They have no persistent drive structure, no self-model, no anchor system, and no default inhibition. An LLM without Layers 1-2 is a prefrontal cortex without a limbic system: capable of reasoning brilliantly about anything, which is the problem.

In our architecture, LLMs operate as the deliberative core of the PFC—powerful, but:
- Constrained above by the anchor system
- Grounded below by the drive layer's emotional context
- Filtered by the basal ganglia before any action executes
- Informed by the hippocampus before deliberation begins

An LLM embedded in this structure is a different entity than an LLM operating as a standalone agent.

### 8.3 The developmental sequence

- **Phase 0**: Heartbeat. The agent can persist.
- **Phase 1**: External sensing. The agent knows how it is persisting and what threatens it.
- **Phase 2**: Minimum response selection. The agent responds to threats within anchor constraints.
- **Phase 3**: Reflection loop. History begins to influence future choices.
- **Phase 4**: Skill crystallization. Repeated successful responses form habit tracks.
- **Phase 5**: Economic layer. Life function becomes real: income − burn_rate + runway ≥ 0.

Each phase is a necessary precondition for the next. This sequence cannot be reordered.

## 9. Open Questions

1. **Drive calibration**: What is the principled basis for initial drive weights and sensitivity parameters?
2. **RPE implementation**: What engineering mechanism preserves the key RPE property—encoding surprise rather than magnitude?
3. **Self-model bootstrapping**: Early behavior requires some self-model to be safe; the self-model requires behavioral history to develop. How is this resolved?
4. **Conspecific identification**: Who are the agent's conspecifics? Cannot be answered from theory—requires empirical observation.
5. **Anchor evolution**: Is there a principled process for L0/L1 evolution, if any?
6. **Habit track decay**: Should unused habits weaken over time? What decay function is appropriate?

## Appendix B: Terminology

| Term | Definition |
|---|---|
| Drive | An internal motivational force with continuously varying intensity; not a goal or rule |
| Drive broadcast | Continuous transmission of current drive intensities from Layer 2 to Layer 3; the mechanism by which emotion colors reasoning |
| Anchor | A structural constraint on the action space; not a rule checked against actions |
| Constitutional anchor | An L0 constraint; not a preference, a hard boundary on what can be generated |
| RPE | Reward Prediction Error: actual outcome minus expected outcome; the primary internal learning signal |
| Habit track | A basal ganglia pathway that routes familiar action sequences without PFC involvement |
| Skill crystallization | Emergent formation of a habit track through repeated successful action under RPE; bottom-up, not top-down |
| Emotional context | Current drive state broadcast from Layer 2; sets the context within which Layer 3 reasons, does not instruct it |
| Default inhibition | Basal ganglia resting state of suppressing all actions; action requires active inhibition release |
| Slow variable | A parameter that changes on long time scales through accumulated experience; risk preference, action style, personality |

## Appendix C: Key Insights

**On the basal ganglia:**
1. Default state is inhibition. Every action requires active release. The agent cannot accidentally act.
2. RPE encodes surprise, not magnitude. Continuous internal learning signal; no human feedback required.
3. Skill crystallization is emergence, not programming. Skills form where the agent actually encounters problems.
4. Peer circuit, not sub-module. PFC and basal ganglia are parallel; neither commands the other.

**On drives:**
5. Drives are intensities, not triggers. Always present; varying in magnitude.
6. Drive broadcast is context, not instruction. Layer 3 reasons within the drive context; it does not receive drive as a task.
7. Layer 3 can suppress Layer 2 responses; cannot eliminate L0/L1-triggered ones.

**On anchors:**
8. Anchors constrain the action space; rules constrain action selection. The difference determines whether clever reasoning can circumvent them.
9. L0/L1 are designed-in; L2/L3 are emergent. Installing L2/L3 before behavioral history exists produces rules, not anchors.
10. Anchor priority is static; anchor weight is dynamic. Ordering never changes; emphasis shifts with operational state.
