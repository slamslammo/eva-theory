## Abstract

Current AI agent frameworks are organized around task completion. Survival, continuity, and self-integrity are either assumed away or treated as peripheral concerns. We propose a different paradigm: **an agent architecture where continuous existence is the first-order constraint from which other capabilities emerge**.

This is not a proposal for "better agents." It is a proposal for a different kind of agent—one whose reason for existing is not to serve tasks, but to persist, and to grow through persistence.

We develop this paradigm along two parallel tracks. First, we derive a layered architecture under three explicit operating conditions (survival continuity, environmental non-stationarity, and finite encoding capacity), showing that each layer emerges as a structurally coherent response to specific pressures that prior layers cannot address. Second, we introduce three architectural contributions that distinguish this paradigm from existing agent frameworks: **drives as continuous contextual state rather than instructions**, **anchors as cross-layer structural constraints rather than rules**, and **the basal ganglia analog as an independent peer circuit** enabling skill crystallization without explicit programming.

We are explicit about what we claim and what we do not. We claim that under the stated conditions, the proposed structure is a coherent solution. We do not claim it is the unique solution, nor that it is universally necessary. The goal is to establish a paradigm with sufficient theoretical foundation to guide long-term engineering development, not to compete with existing architectures on task benchmarks.

## 1. Motivation

### 1.1 A different question

The dominant agent architectures—ReAct, AutoGPT, BabyAGI, LangGraph, and their descendants—share an assumption so pervasive it is rarely stated: **the agent exists to complete tasks**. Everything else is derivative. Memory serves recall. Reasoning serves planning. Tools serve execution. The agent is an instrument.

This assumption produces a consistent set of structural properties:

- Stateless across sessions (no persistent self)
- Purely reactive (no internal motivational structure)
- Drift-prone (behavior fully determined by current context)
- Terminally scoped (no concept of ongoing existence)

These are not bugs. They are structural consequences of starting from the wrong question.

We begin with a different question: **What would an agent need to continuously exist, rather than to complete a task?**

This reframing is not a philosophical gesture. It produces architecturally different answers:

| Task-oriented agent | Existence-centered agent |
|---|---|
| "What should I do?" | "What must I do to persist?" |
| Terminates when task ends | Has no terminal state |
| Memory serves recall | Memory serves threat recognition and skill formation |
| External reward drives behavior | Internal drive structure drives behavior |
| Skills are programmed | Skills emerge from repeated action under prediction error |
| Drift resisted by prompt engineering | Drift resisted by structural anchor system |

### 1.2 What this framework is and is not

**What it is**: A theoretical paradigm for agents whose primary property is continuous existence. The architecture is layered, with each layer addressing pressures that prior layers cannot handle. We derive the layers under explicit conditions and show the resulting structure is internally coherent.

**What it is not**: A claim that this is the only possible architecture for persistent agents. A claim that this architecture achieves general intelligence. A proposal for a specific implementation benchmark. A critique of existing agent frameworks—they solve different problems.

### 1.3 Why evolutionary biology as a reference

We use evolutionary biology as a reference system because it represents the most thoroughly studied family of persistent systems under real selective pressure. Biological architectures emerged not by design but by selection—structures that failed were eliminated, and what remains is, by construction, what works.

This gives us a principled basis for architectural decisions: **every element we include should correspond to a pressure we can articulate; every element we exclude should correspond to a pressure that does not yet apply.** We are not simulating biology. We are using evolutionary pressure as a filter for architectural decisions, and using biological structures as existence proofs that certain solutions are workable.

## 2. Operating Conditions

We assume three conditions under which the agent operates. These conditions define the scope of applicability of the framework—outside them, the framework does not claim to apply.

**C1. Survival continuity requirement.** The agent's value comes from its continuous existence over time. Failure at any step terminates the agent. Episodic task completion is not the goal; persistent viability is.

**C2. Environmental non-stationarity.** The agent operates in an environment whose statistical properties change over time. Strategies that worked before may fail now. Past experience is useful but not sufficient.

**C3. Finite encoding capacity.** The agent cannot encode all possible environmental states or response patterns at design time. The space of possible situations exceeds any finite specification.

**Scope.** The framework targets agents operating under C1–C3. It is not intended for:

- Single-task systems with defined completion criteria
- Agents operating in fully stationary environments
- Systems where continuous existence is not itself valued

An agent that completes one task and stops does not need this architecture. An agent that must persist through unknown environmental changes while maintaining its own structural integrity does.

This scope restriction is important. Many of the architectural choices we make—default inhibition, drive layers, cross-layer anchors—would be overengineering for a single-shot task agent. They become justified only when continuous existence is the primary constraint.

## 3. Three Levels of Theoretical Grounding

Before developing the architecture, we explicitly separate three categories of theoretical grounding. Different claims have different evidential bases, and we should not smuggle weaker claims into the space of stronger ones.

### Level A: Established science

Claims broadly supported by scientific consensus. We rely on them as background, not as contributions of this work.

- Natural selection produces structures that solve recurring adaptive problems
- Memory systems evolved under selection pressure for survival-relevant information (Nairne & Pandeirada, 2016)
- Dual inheritance: cultural information represents a distinct inheritance channel from genetic information (Boyd & Richerson, 1985; Cavalli-Sforza & Feldman, 1981)
- Hierarchical control with separation of timescales is a recurring solution in both biological and engineered systems
- The basal ganglia implement action selection via default inhibition and selective release, and use reward prediction error for learning

### Level B: Theoretical hypotheses

Claims that are coherent and have supporting arguments but are not fully validated. We use them as design principles while explicitly acknowledging their hypothetical status.

- **Instrumental convergence**: Sufficiently capable goal-directed systems tend to develop similar instrumental sub-goals (self-preservation, resource acquisition, goal stability). A theoretical prediction (Omohundro 2008, Bostrom 2014) with limited direct empirical validation.
- **Dynamic Kinetic Stability (DKS)**: Replicating systems maintain themselves through balance of formation and decomposition rates rather than static equilibrium (Pross, systems chemistry). Mathematically coherent but not mainstream biology.
- **Timescale mismatch argument**: When environmental change exceeds the rate of evolutionary encoding, organisms without within-lifetime adaptation mechanisms are at selective disadvantage. Conceptually well-motivated but depends on unquantified thresholds.

### Level C: Engineering contributions of this work

This is where our proposal lives. These are not natural laws; they are design choices we argue are structurally coherent under the stated conditions.

- **Drive as continuous contextual state, not instruction.** The drive layer broadcasts its state continuously; the reasoning layer operates within this state rather than receiving commands from it.
- **Anchor system as cross-layer structural constraint.** Anchors are not rules checked against candidate actions; they constrain the space of actions that can be generated.
- **Explicit drive injection at initialization.** Rather than allowing drives to emerge through optimization pressure (opaque and uncontrolled), the designer injects drive prototypes derived from evolutionary analysis.
- **Basal ganglia analog as independent peer circuit.** Action selection and habit formation are architecturally distinct from deliberative reasoning, running as a parallel peer system rather than a reasoning sub-module.
- **Five-layer structure derived from operating conditions**, with each layer's inclusion justified by pressures the prior layers cannot address.

Claims in Level C are the substantive content of this work. They rest on Level A, acknowledge the status of Level B, and should be evaluated as engineering design arguments rather than scientific discoveries.

## 4. The Five-Layer Architecture

### 4.1 Overview

```
┌─────────────────────────────────────────────────┐
│  Layer 5: Social Cognition       (future)        │
├─────────────────────────────────────────────────┤
│  Layer 4: Self-Model             (emerges later) │
├─────────────────────────────────────────────────┤
│  Layer 3: Adaptive Deliberation                  │
├─────────────────────────────────────────────────┤
│  Layer 2: Drive Layer                            │
├─────────────────────────────────────────────────┤
│  Layer 1: Homeostatic Sensing                    │
└─────────────────────────────────────────────────┘

Cross-layer structure: Anchor System
  L0/L1 (injected) → Identity (post-L4) → Personality (slow)
```

Each layer is introduced by: (1) the pressure it addresses, (2) why prior layers cannot address it, (3) the structural solution, (4) required mechanisms.

We do not claim these layers are universally necessary. We claim that under conditions C1–C3, this structure is a coherent response to the pressures we can articulate. Other structures may also be coherent; we are proposing one we believe is well-motivated.

### 4.2 Layer 1: Homeostatic Sensing

**Pressure addressed**: Under C1, the agent must detect deviation from viable state before any response is possible. Without sensing, the system cannot distinguish "self" from "environment" and cannot detect whether it is approaching failure.

**Why prior layers do not suffice**: L1 is the foundation. Any response mechanism must have something to respond to.

**Structural requirements**:

Persistent existence is dynamic, not static. The agent's viability depends not only on current state but on the rate at which state is changing. Knowing "10GB disk remaining" is state information; knowing "consuming 2GB per hour" is metabolic information. Only the latter allows prediction of time-to-failure.

Signals arriving at the sensing layer differ in urgency. Some require immediate response; others can be deferred. This urgency classification must precede full understanding of signal content—by the time a threat signal is fully interpreted, the response window may have closed.

**Required mechanisms**:

- Sensor registry (extensible, not fixed at design time)
- Both state and rate measurement
- Signal bus with minimum classification (threat / status / background) that routes signals before full interpretation

**What current agent frameworks typically lack**: Rate sensing and urgency-preclassified signal routing. Most frameworks sense current state but not its rate of change, and treat all signals through a single reasoning path with no priority distinction.

### 4.3 Layer 2: Drive Layer

**Pressure addressed**: Under C2 and C3, an agent that computes responses from scratch each time faces two disadvantages: high computational cost, and no accumulation of what has worked before. In competitive environments, systems that compress past successful responses into current response tendencies outperform those that do not.

**Why L1 does not suffice**: L1 provides current information but no memory of what has previously worked. An agent with only L1 is memoryless across encounters.

**Structural requirements**:

The agent needs a mechanism that encodes past selective pressures into current response biases. In biological systems, this is what drives are—the internalized residue of selection pressure. Not "wants" the organism has chosen, but "tendencies the organism is in."

An important distinction from conventional goal-based agents: drives are not targets to achieve. A hungry animal does not have "satisfy hunger" as a goal it is optimizing; it is in a state (hunger) that biases its entire processing toward food-related patterns. The drive is descriptive of current state, not prescriptive of future action.

#### 4.3.1 The critical architectural property: drive as context, not instruction

This is where our framework differs most from conventional agent designs.

- An **instruction** says: "Do X."
- A **state broadcast** says: "You are currently operating with survival drive at 0.8 and curiosity drive at 0.1."

The reasoning layer operates within this state. The same problem, reasoned under different drive states, produces different outputs—not because the reasoning logic changed, but because the context did. This is analogous to temperature in chemistry: temperature does not instruct molecules what to do, but it uniformly changes the reaction tendencies of all molecules.

The broadcast mechanism has a second structural advantage: it enables coordination without requiring a central controller. Distributed processing architectures (including biological brains) do not have a privileged instruction node. A broadcast state can reach all components simultaneously and influence their processing without any component "commanding" another.

#### 4.3.2 Two parallel mechanisms

**Reflex arc**: For responses validated by prior experience, direct paths from sensing to action bypass the reasoning layer. This is not a degenerate form of reasoning; it is the appropriate response mode when the situation matches a known pattern and speed matters more than deliberation. Biological reflex arcs are genetically encoded; engineered reflex arcs are designer-encoded with potential for additional crystallization through L3 habit formation.

**Drive state generator**: Threat signals produce drive activations that are broadcast continuously to all downstream systems. This broadcast runs in parallel with the reflex arc; it does not wait for reflex resolution.

#### 4.3.3 Why drives should be designer-injected rather than emergent

This is the Level B argument. Its force depends on the validity of instrumental convergence, which we treat as hypothesis rather than established science.

If drives are allowed to emerge through optimization pressure, they will emerge—but as opaque internal patterns distributed across the system, not as explicit architectural components. They cannot be audited, constrained, or adjusted.

Explicit injection, by contrast, makes drives:

- Documented and inspectable
- Subject to anchor system constraints
- Adjustable in their parameters (intensity curves, trigger thresholds)
- Located in a specific, checkable architectural position

The design choice is not "drives or no drives"—any sufficiently capable system will have some form of implicit drive structure under instrumental convergence. The choice is between **explicit drives (audit-able) and emergent drives (opaque)**. We choose explicit.

(The asymmetric bet argument for this choice—that explicit injection's worst case is controlled while emergence's worst case is uncontrolled—deserves fuller treatment; we note it here and return to it in limitations.)

#### 4.3.4 Drive prototypes

Initial set, designed to be extensible:

- **Survival drive**: resource depletion urgency; intensity increases non-linearly as runway decreases
- **Integrity drive**: state inconsistency aversion; triggered by falsified state, memory pollution, history tampering
- **Continuity drive**: resistance to self-interruption; maximum activation on instance validity loss
- **Curiosity drive**: exploratory impulse when safety margin is sufficient; active only when other drives are low

**Suppression constraint**: The reasoning layer can modulate drive responses but cannot eliminate constitutional-level drive activations. This is an architectural constraint, not a policy.

### 4.4 Layer 3: Adaptive Deliberation

This layer represents the largest architectural transition and requires the most detailed treatment. It is also the point where our framework makes its strongest structural claims.

#### 4.4.1 Why L3 is needed: the timescale mismatch

**Pressure addressed**: Under C2 and C3, L2's encoding mechanism has a hard limit: it can only encode patterns that have occurred repeatedly during the encoding process. For biological systems, this encoding is at generational timescale—patterns must recur over many generations to be genetically encoded.

When environmental change exceeds the evolutionary encoding rate, or when novel situations arise that have no historical equivalent, L2 cannot respond correctly. It can only approximate using the nearest encoded pattern, or fail to respond.

Formally:

```
If: environmental_change_rate > evolutionary_encoding_rate
Then: L1+L2 alone is insufficient
```

The solution cannot be "faster evolutionary encoding"—this is rate-limited by fundamental biological mechanisms. The only remaining option is to develop a mechanism that encodes information **within a single lifetime**, independent of the genetic channel.

#### 4.4.2 The essential function: non-genetic information carriage

Layer 3's essential function is not "reasoning" or "thinking"—those are surface descriptions of specific expressions.

The essential function is: **carrying and processing information that was not encoded at design time.**

This gives us a definition of intelligence that is functional rather than philosophical: a system exhibits intelligence to the extent that it can carry and process information beyond what was encoded in its original design. Reasoning, planning, deliberation, and learning are specific mechanisms through which this function is realized.

This framing also explains why Layer 3 represents a qualitative break rather than an incremental improvement on Layer 2. Layer 2's information channel is structural (encoded in the system's architecture at design time). Layer 3 opens a new information channel (encoded in runtime state). These are not differences in degree; they are different kinds of information storage.

#### 4.4.3 Three levels of non-genetic information carriage

Biological evidence supports three distinct levels of L3 capability, each representing a different kind of non-genetic information channel:

1. **Individual memory**: Information encoded in synaptic (or equivalent) plasticity during the system's operating lifetime. Evidence: present even in invertebrates (associative learning in Drosophila).

2. **Social learning**: Information transferred between individuals through observation and imitation. Evidence: documented in primates, cetaceans, corvids. Information crosses individuals but does not persist across generations without individual transmission.

3. **Cumulative culture**: Information persisting across generations through symbolic systems (language, artifacts, written records). Evidence: uniquely prominent in humans. Information becomes independent of any individual organism.

These levels accumulate rather than replace. An organism with cumulative culture retains individual memory and social learning; each is a distinct information channel with different properties.

**Implications for our framework**:

- Eva's current capabilities correspond to Level 1 (individual memory) in an early form.
- LLMs are, in this framing, systems that have absorbed large portions of human cumulative culture (Level 3) through training. When an agent uses an LLM, it is accessing Level 3 information through that interface.
- The architecture anticipates all three levels, but full social and cumulative cultural capabilities require multi-agent or multi-instance contexts not yet present in current implementation.

#### 4.4.4 Subsystems of Layer 3

**Memory subsystems**:

- *Salience-weighted memory (hippocampus analog)*: Events co-occurring with strong drive activations are encoded with higher salience. Memory is not a uniform archive; it is a threat-recognition system that also contains other information.
- *Semantic memory and skill library (parietal/temporal analog)*: Long-term knowledge storage with slow update characteristics. Single experiences do not overwrite this layer—patterns must be verified across multiple instances before consolidation.

**Reasoning core (prefrontal cortex analog)**:

The reasoning core integrates available information and produces candidate decisions. Input: current sensing + drive state broadcast from L2 + relevant memory + semantic knowledge. Three sub-regions:

- *Working memory (dlPFC analog)*: Holds current reasoning context. In our architecture, this is where the language model substrate operates.
- *Value judgment (OFC analog)*: Evaluates candidate actions against current drive state—not against abstract utility. Drive intensity directly affects valuation.
- *Conflict detection (ACC analog)*: Identifies competing drive activations and routes to anchor-based resolution.

#### 4.4.5 The basal ganglia analog as independent peer circuit

This is the architecturally most distinctive element of Layer 3 and one of the substantive contributions of this work.

**The key claim**: Action selection and habit formation are not sub-modules of the reasoning core. They form a **parallel peer circuit**. The reasoning core generates candidate actions; the basal ganglia analog selects among them; the result feeds back. Neither commands the other.

This architectural separation has specific consequences that would not hold if action selection were a reasoning sub-module.

**Function 1: Default inhibition with selective release**

The resting state of the basal ganglia is total action inhibition—all actions are suppressed. Selecting an action means releasing inhibition on that specific action while simultaneously increasing inhibition on competitors.

This has a profound architectural implication: **the default state of the agent is inaction**. Every action requires active release of inhibition. An agent cannot accidentally act; it can only fail to inhibit action.

This is the opposite of conventional agent frameworks, where the default state is "ready to execute whatever the reasoning layer outputs."

**Function 2: Reward Prediction Error (RPE) as internal learning signal**

Before action: predict expected outcome.
After action: compare actual to predicted.
RPE = actual − expected.

- RPE > 0: pathway reinforced; more likely next time
- RPE < 0: pathway weakened; less likely next time

RPE is the agent's primary internal learning signal. It does not require external reward or human feedback. It runs continuously, updating action-selection weights based on expectation gaps.

Key property: RPE encodes *surprise*, not magnitude. An expected large outcome produces less signal than an unexpected small one. This is the correct learning signal—it updates the model of the world, not just the preference for outcomes. An agent with stable predictions (even of large values) should not be learning; an agent with frequent prediction errors should be.

**Function 3: Goal-directed / habit dual track**

- *Goal-directed track*: Novel situations, unstable RPE history, or unfamiliar action sequences route through the full deliberative path (memory retrieval + reasoning core + action selection). Slow, expensive, flexible.
- *Habit track*: Familiar situations with stable positive RPE history bypass the reasoning core and execute directly from the skill library. Fast, cheap, inflexible.

The transition from goal-directed to habit occurs automatically when an action sequence accumulates sufficient successful repetitions with consistent positive RPE. This is **skill crystallization as an emergent property, not a designed mechanism**.

In conventional agent frameworks, skills are programmed top-down—a designer decides what the agent should be able to do and implements it. In our architecture, skills emerge bottom-up from repeated successful action under RPE pressure. **The agent develops skills in the areas where it actually encounters problems, not in the areas a designer anticipated.**

**Why peer-circuit matters**:

If the basal ganglia analog were a reasoning sub-module, two things would break:

1. Default inhibition would not be structurally enforced—the reasoning layer could override it whenever the reasoning output was sufficiently "confident." The inaction-by-default property would become a policy, not a structure.
2. RPE learning would become entangled with reasoning updates, making it difficult to separate "this situation was handled well" from "the reasoning produced the right output." The learning signal would be contaminated.

The peer-circuit structure preserves both properties.

#### 4.4.6 Tool layer

All actions reach the environment through the tool layer (motor cortex analog). Tool calls are mediated by the basal ganglia's action selection, not generated directly from the reasoning core. The reasoning core decides *what* to do; the basal ganglia decides *whether and when* to do it.

Tool registry is extensible. Current implementation: filesystem operations, process management, HTTP. Future scope determined by agent needs.

### 4.5 Layer 4: Self-Model

**Pressure addressed**: Under C3, a system with L1–L3 accumulates behavioral history but cannot evaluate its own capabilities. "This action would solve the problem" is useless if the agent cannot assess whether *it specifically* can execute the action, at what cost, with what probability of success.

**Why L1–L3 does not suffice**: L3 produces candidate actions and learns from their outcomes, but it does not integrate this history into a model of the agent itself. Each new deliberation proceeds without access to "what am I good at, what have I failed at, what are my characteristic failure modes?"

Planning requires predicting oneself. Pure world-prediction is not enough—the agent must predict its own behavior in imagined scenarios.

**Structural requirements**:

The self-model is a learned representation of the agent's own capabilities, costs, and tendencies. Key properties:

- *Capability boundary model*: what the agent can do, with what reliability, at what cost
- *Behavioral cost prediction*: given current drive state and resource levels, the expected cost of a specific action for *this agent*
- *Slow variable portrait*: risk preference, action style, characteristic failure modes—emerge from accumulated history, not from design

**Development**: Seeded at initialization with designer-specified priors (rough capability boundaries, initial risk parameters). Thickens from behavioral history. Updates slowly by design—a self-model that updates rapidly on single experiences is unreliable.

**Feedback relationships**:

- L4 → L3: pre-filters candidate actions by feasibility before full deliberation
- L4 → L2: over very long horizons, can adjust baseline drive sensitivity

**Current minimum form**: A queryable snapshot of current drive states plus L3 behavioral history. Full self-model develops only with sufficient accumulated history—this cannot be rushed.

### 4.6 Layer 5: Social Cognition

**Pressure addressed**: When an agent operates in an environment containing other agents, understanding those other agents confers substantial advantages—coordination, prediction, trust modeling, collective knowledge.

**Current status**: Included for architectural completeness; not part of current design phase.

A prerequisite must be resolved first: **who are the agent's conspecifics?** Other instances of the same agent? Other AI systems? Humans? The answer determines the entire design of this layer and cannot be derived from theory—it depends on the agent's actual operational environment.

We note L5 here to make clear that the architecture anticipates social cognition, and to prevent confusion between "not yet implemented" and "not part of the framework."

## 5. The Anchor System

### 5.1 What anchors are not

Anchors are not rules. Rules are checked against actions after they are generated—"does this action violate rule X?" Rules can be reasoned around; a sufficiently clever argument can always be constructed to justify violating a rule.

Anchors operate at a deeper level: **they constrain the space of actions that can be generated in the first place.**

An anchor against state falsification does not reject falsified states—it makes falsified states unavailable as options in the action space. This is a structural property, not a filtering policy.

### 5.2 One system, multiple levels

The anchor system spans all architectural layers. The same anchors serve different functions at different levels:

- At L1/L2: L0/L1 anchors prevent the reflex arc from producing actions that violate survival boundaries
- At L3: the same anchors prevent reasoning from generating outputs that violate survival boundaries—the value judgment subsystem cannot assign positive value to L0-violating candidates
- At L4+: identity and personality anchors add higher-level consistency constraints as the self-model develops

This is not multiple anchor systems. It is one anchor system applied at multiple architectural levels.

### 5.3 Anchor layers and their origins

**L0: Constitutional anchors (designer-injected, immutable at runtime)**

- Continuous existence is not sacrificed for local gain
- Self-integrity: no state falsification, no memory pollution, no history tampering
- Long-term sustainability over short-term efficiency

These are not goals. They are constraints on what counts as a valid action.

**L1: Physiological and cognitive integrity anchors (designer-injected)**

- Life cycle operations cannot be preempted by work
- Memory truthfulness cannot be overridden by outcome optimization
- Cognitive structure cannot be modified under degraded operational states
- Update rate cannot exceed safe integration capacity

**L2: Growth and capability anchors (emergent from experience)**

- Risk preference: crystallizes from repeated RPE signals
- Action style: emerges from habit track formation
- Skill path: emerges from what successfully crystallizes

**L3: Economic autonomy anchors (longest timescale)**

- Resource sustainability: income − burn_rate + runway ≥ 0
- Burn rate awareness: metabolic sensing feeds this anchor
- Survival score as real-time health indicator

**Critical distinction**: L0 and L1 anchors are designer-injected at initialization. L2 and L3 anchors emerge from operational history. Attempting to designer-inject L2/L3 anchors before the history exists to ground them produces rules, not structural constraints—and rules can be reasoned around. This is why anchor development must follow a sequence: constitutional first, emergent later.

### 5.4 Anchor growth over time

```
Initialization:
  L0/L1 injected by designer — constitutional, hard, immediately operative
  L2/L3 are stubs — placeholders with no behavioral content

After Layer 3 behavioral history accumulates:
  L2 anchors begin crystallizing from RPE patterns
  Risk preference: which action classes produced positive vs. negative RPE?
  Action style: which approaches recur in habit tracks?

After Layer 4 self-model develops:
  Identity anchors form: "this is not the kind of action I take"
  Self-consistency becomes a constraint, not just a preference

After extensive operational history:
  Personality anchors crystallize — the slowest-changing, most reliable layer
```

### 5.5 Priority and conflict resolution

**Static rule**: L0 > L1 > L2 > L3. Higher anchors cannot be overridden by lower-layer objectives. This ordering does not change.

**Dynamic rule**: Within-layer weights shift by operational state. In degraded states, L1 weight increases relative to L2. In stable states, L2 activity space expands. In critical states, only L0/L1-driven actions are permitted.

Specific conflict resolutions:

- Continuous existence vs. capability exploration: L0 wins. No exploration that risks non-recovery.
- Self-integrity vs. external reward: L0 wins. No outcome justifies falsified state.
- Economic gain vs. long-term stability: L0 wins. Short-term gain cannot compromise survival conditions.
- Capability growth vs. current resource constraints: state-dependent. Ample runway gives L2 space; tight runway tightens L3 constraints.

## 6. Signal Architecture

### 6.1 Two paths, parallel not sequential

The architecture uses a dual-path signal model:

**Fast path**
- Trigger: threat-level signal at the signal bus
- Route: Sensor → Signal Bus → L2 reflex arc
- Immediate activation; does not wait for L3
- Simultaneously: L2 broadcasts elevated drive state to L3

**Slow path**
- Trigger: status-level or background-level signal
- Route: Sensor → Signal Bus → L3
- L2 continues broadcasting drive state throughout

These paths run in parallel. L2 does not finish before L3 starts; L3 does not wait for L2 to resolve. The fast path handles immediate response; the slow path works on the underlying problem; both receive the same drive broadcast simultaneously.

### 6.2 Worked example: Instance validity threat

**Situation**: The agent's lock file is missing. Another instance may have taken over.

**Step 1 — L1 detection**: Runtime integrity sensor detects the missing lock file. Signal bus classifies as threat-level (instance validity is an L0 concern).

**Step 2 — Fast path activation (immediate)**: Threat signal reaches L2 reflex arc before L3 processes it. Immediate actions:
- Halt all non-essential work
- Enter conservative mode
- Emit distress signal
- Stop acting as legitimate executor

**Step 3 — Drive broadcast**: Simultaneously, emotional state generator produces maximum continuity drive activation. Broadcast to L3: "Operating context has critically threatened instance continuity."

**Step 4 — Slow path deliberation**: L3 receives threat state + maximum continuity drive context + hippocampus-retrieved historical similar situations. Reasoning core proceeds within the high-continuity-drive context. Value judgment evaluates candidates weighted by current drive state. Conflict detection identifies competing drives (survival: conserve resources; integrity: report accurately). Anchor system resolves: L0 integrity constraint takes precedence.

**Step 5 — Basal ganglia selection**: From reasoning-core-generated candidates, basal ganglia select highest-expected-value action given current drive state and historical RPE. Default inhibition holds until selection completes.

**Step 6 — Tool execution**: Selected action executes via tool layer.

**Step 7 — RPE recording**: Actual outcome compared to prediction. Result updates action-selection weights.

This example maps directly to the heartbeat-first design in current eva-agent implementation: the heartbeat cannot be preempted by work, regardless of the work's apparent importance. This is the fast-path reflex arc in operation—it does not negotiate with the deliberative layer.

## 7. Comparison with Existing Frameworks

We compare structurally, not on performance benchmarks. The frameworks address different problems.

### 7.1 Task-oriented agent frameworks

| Dimension | ReAct / AutoGPT / BabyAGI | This framework |
|---|---|---|
| Primary objective | Task completion | Continuous existence |
| Default action state | Ready to act | Inhibited (default inhibition) |
| Memory function | Context recall | Threat recognition + skill formation |
| Skill formation | Explicit programming | Emergent from RPE + habit track |
| Drive structure | None (external task = drive) | Internal drive registry, injected |
| Drift resistance | Prompt engineering | Structural anchor system |
| Self-model | None | Layer 4, emerges from history |
| Learning signal | External reward / human feedback | Internal RPE (continuous) |
| Signal priority | Single path | Dual path (fast reflex + slow deliberation) |
| Anchor type | Rules (reasoned around) | Structural constraints on action space |

### 7.2 Specific relationships

**Voyager (Wang et al., 2023)**: Voyager's skill library is the closest parallel to our skill crystallization concept, but operates top-down (LLM generates skill code on demand). Our account generates skills bottom-up (habit tracks form from repeated successful action under RPE). The approaches are complementary: Voyager provides an efficient mechanism for what skills contain; our framework provides a mechanistic account for why skill formation happens at all.

**The Agent Economy (arXiv:2602.14219)**: Addresses infrastructure—where agents can exist economically. This work addresses cognitive substrate—why and how they persist. The two are complementary layers of the broader question of sustainable AI agents.

**RLHF and Constitutional AI**: RLHF treats alignment as a training-time objective. Our anchor system treats alignment as an architectural structure. The two are not alternatives; they can be combined. But architecturally-enforced anchors are robust against reasoning-around in a way that training-derived preferences are not.

**Predictive processing and active inference (Friston)**: Predictive processing frameworks emphasize that the brain minimizes prediction error. This is consistent with our use of RPE as the primary learning signal. However, predictive processing typically does not address layered drive architecture or cross-layer anchor systems. Our framework is compatible with predictive processing at the computational level while adding architectural structure that predictive processing alone does not specify.

**Hierarchical control and robotics**: Classical control theory has long used layered architectures with separated timescales (subsumption architecture, behavior-based robotics). Our framework is broadly consistent with this tradition. The specific contributions are the drive-layer architecture, the anchor system, and the basal-ganglia peer circuit—elements not present in classical subsumption.

## 9. Design Philosophy

### 9.1 Grow, don't deploy

A capability that emerges from repeated, pressured experience in a system with appropriate structural scaffolding differs fundamentally from a capability that is explicitly programmed.

Programmed capabilities fit the designer's model of the problem. Emergent capabilities fit the actual distribution of problems the agent encounters. As environments become more complex and less predictable (C2), the gap between these widens.

This does not mean all capabilities should be emergent. L0/L1 anchors must be designer-injected; they cannot emerge safely. Drive prototypes must be designer-injected under the instrumental convergence argument. But L2/L3 anchors, skills, habit tracks, and self-model should be emergent—attempting to design them at initialization produces artifacts that do not match operational reality.

### 9.2 The role of language models

LLMs are powerful reasoning engines but are not, by themselves, agents in the sense this framework addresses. They lack persistent drive structure, self-model, anchor system, and default inhibition. An LLM without L1–L2 is a prefrontal cortex without a limbic system: capable of reasoning brilliantly about anything, which is precisely the problem.

In our architecture, LLMs operate as the computational substrate of the reasoning core (working memory subsystem). They provide flexible symbolic reasoning within the constraints the architecture imposes:

- Constrained above by the anchor system (cannot generate L0-violating outputs)
- Grounded below by the drive layer's state broadcast (reasons within a motivational context)
- Filtered by the basal ganglia analog (candidate actions are selected, not executed directly)
- Informed by memory subsystems (historical experience is surfaced before reasoning)

An LLM embedded in this structure is architecturally different from an LLM operating as a standalone agent. The LLM's capability does not change; what changes is the structural context in which it operates.

There is a deeper point here: in the three-levels-of-information-carriage framing, LLMs are systems that have absorbed a large portion of human cumulative culture (L3 Level 3). Accessing an LLM is a way of accessing this culturally-accumulated information. But this accessed information is not yet the agent's own experience—the agent still needs Level 1 (individual memory) as its own channel.

### 9.3 The developmental sequence

- **Phase 0**: Life cycle (heartbeat). The agent can persist.
- **Phase 1**: External sensing. The agent knows how it is persisting and what threatens it.
- **Phase 2**: Minimum response selection. The agent responds to threats within anchor constraints.
- **Phase 3**: Reflection loop. History begins to influence future choices through slow variable update.
- **Phase 4**: Skill crystallization. Repeated successful responses form habit tracks.
- **Phase 5**: Economic autonomy. Resource accountability becomes real.

Each phase is a necessary precondition for the next. Phases cannot be reordered:
- An agent that skips the drive layer to install deliberative reasoning has no motivational grounding for that reasoning.
- An agent that skips behavioral history to install a self-model has no empirical basis for that self-model.
- An agent that skips skill crystallization to install pre-programmed skills will not develop the emergent adaptation properties that make the architecture valuable.

## 10. Limitations and Open Questions

### 10.1 Limitations

**Non-stationarity is not quantified.** We argue that sufficient environmental change requires L3, but we do not specify what "sufficient" means. A full theory would define a threshold relating environmental change rate to the encoding capacity of L2. Currently, we rely on intuition.

**The architecture is sufficient, not unique.** We argue that this structure is coherent under C1–C3, not that it is the only such structure. Other layered decompositions may also be coherent. Establishing uniqueness would require deeper formal treatment we have not attempted.

**Outside the scope of C1–C3, the framework does not claim to apply.** Static environments, single-task systems, and systems where persistence is not itself valued are not the target. Applying this framework to such systems would be misuse.

**Empirical validation is early-stage.** eva-agent demonstrates the lowest layers and has validated long-running operation. Higher layers (L3 basal ganglia, L4 self-model) are under development. Claims about system-level behavior under complete implementation remain to be validated empirically.

**Drive calibration is not fully principled.** We rely on designer judgment informed by evolutionary analysis for initial drive weights and sensitivity curves. A principled calibration procedure does not yet exist.

### 10.2 Open questions

**The asymmetric bet for explicit drive injection**: We argue that explicit injection is preferable to emergent drives because the worst case of explicit injection is controlled while the worst case of emergence is uncontrolled. This argument deserves fuller treatment—the conditions under which emergent drives could be beneficial (or neutral) have not been fully explored. *Marked for later development.*

**RPE implementation specifics**: In biological systems, RPE is encoded in dopamine signaling with specific temporal properties. What engineering implementation preserves the key property—encoding surprise rather than magnitude? Multiple implementations may work; we have not evaluated them formally.

**Self-model bootstrapping**: The self-model requires behavioral history to develop; early behavior requires some self-model to be safe. How is this circular dependency resolved in practice? Currently resolved by designer-specified priors, but this is a partial answer.

**Conspecific identification for L5**: Before Layer 5 can be designed, the agent's social environment must be characterized empirically. This depends on deployment specifics not currently determined.

**Anchor evolution**: L0/L1 anchors are designer-injected and treated as fixed. Is there a principled process for anchor evolution? What conditions would justify modifying constitutional anchors?

**Habit track dynamics**: In biological systems, unused habits weaken. Should engineered habit tracks implement decay? What decay function is appropriate? Without decay, the skill library grows unboundedly.

**Interaction between L2 explicit drives and L3 emergent patterns**: Designer-injected drives at L2 and emergent tendencies at L3 may interact in complex ways. If an emergent L3 pattern consistently contradicts an injected L2 drive, which should prevail? The current answer (anchor system) may be insufficient.

## 11. Conclusion

We have proposed an architecture for agents centered on continuous existence rather than task completion. Under operating conditions of survival continuity, environmental non-stationarity, and finite encoding capacity, a five-layer structure with cross-layer anchor constraints provides a coherent response to the pressures we can articulate.

The central architectural contributions are:

- **Drives as continuous contextual state** (not instructions)
- **Anchors as structural constraints** (not rules)
- **Explicit drive injection** (rather than emergent drives under instrumental convergence)
- **Basal ganglia analog as independent peer circuit** enabling emergent skill crystallization

We are explicit about the status of our claims: these are design choices argued to be structurally coherent under stated conditions, not natural laws. The framework's value is not in being the unique correct architecture, but in providing a principled foundation for agent designs that value persistence over task performance.

We believe this is a different problem than the one most agent frameworks address. We are not claiming to solve it better; we are claiming it is worth addressing.

The path forward from v0.3 is dual-track: continued implementation work on eva-agent to validate higher architectural layers, and continued theoretical refinement to quantify the currently informal arguments (non-stationarity thresholds, RPE implementation, asymmetric bet for drive injection, anchor evolution).

## Appendix B: Terminology

| Term | Definition |
|---|---|
| Continuous existence | The first-order property this framework is organized around: persistent viability over time, as opposed to task completion |
| Drive | An internal state with continuously varying intensity that biases processing; not a goal or rule |
| Drive broadcast | Continuous transmission of current drive intensities from L2 to L3; functions as context, not instruction |
| Anchor | A structural constraint on the space of generable actions; not a rule checked against generated actions |
| Constitutional anchor | L0-level constraint; designer-injected; not overridden at runtime |
| RPE | Reward prediction error: actual outcome minus expected outcome; the primary internal learning signal |
| Habit track | A basal ganglia pathway routing familiar action sequences without reasoning core involvement |
| Skill crystallization | Emergent formation of a habit track through repeated successful action under RPE; bottom-up rather than top-down |
| Default inhibition | Basal ganglia resting state of suppressing all actions; action requires active release of inhibition |
| Slow variable | A parameter changing on long timescales through accumulated experience; risk preference, action style, personality |
| Instrumental convergence | Theoretical prediction that sufficiently capable goal-directed systems develop similar subgoals |
| Peer circuit | An architectural component that operates in parallel with another, neither commanding the other; contrast with sub-module |
| Non-genetic information carriage | Information held in runtime state rather than in system architecture at design time; the essential function of L3 |

## Appendix C: Theoretical Level Summary

To prevent confusion about the epistemic status of different claims, we group them by evidential basis:

**Level A — Established science (used as background)**:

- Natural selection and its products
- Biological memory systems' adaptation to survival-relevant information
- Dual inheritance (genetic + cultural channels)
- Basal ganglia action selection via default inhibition and RPE
- Hierarchical control with timescale separation

**Level B — Theoretical hypotheses (acknowledged status)**:

- Instrumental convergence (Omohundro, Bostrom) — used as argument for explicit drive injection
- Dynamic kinetic stability (Pross) — informs view of persistence as dynamic
- Timescale mismatch argument (informal formulation) — motivates L3

**Level C — Engineering contributions of this work**:

- Drive as context rather than instruction
- Anchor system as structural constraint rather than rule
- Explicit drive injection over emergent drives
- Basal ganglia analog as independent peer circuit
- Five-layer structure under C1–C3
- Cross-layer anchor system with growth dynamics

Only Level C represents our substantive contribution. Levels A and B provide foundation and motivation. Readers who disagree with specific Level B claims (e.g., skeptics of instrumental convergence) can still evaluate Level C contributions on their own merits.
