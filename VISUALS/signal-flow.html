EVA — Signal Flow Overview

A signal-flow view aligned with v0.5. Using the same five-layer architecture as the overview, it shows how urgent and non-urgent signals move through EVA: fast reflex through L1–L2, slower deliberation through L1–L2–L3, drive broadcast into candidate generation, and execution only after mediated release.

architecture-aligned flow view fast vs slow path mediated release
How to read this diagram. Read it as the flow companion to the five-layer overview. The stack stays the same: L5 at the top, L1 at the bottom, anchor spine on the left. Start at L1 sensing, where signals enter and split. Urgent input can take the fast path through L1–L2 and reach execution quickly. Non-urgent input follows the slow path through L1–L2–L3, where candidate generation is anchor-bounded, drive enters as contextual broadcast, and release happens only inside L3's peer circuit.

Visual emphasis

  • Fast path overlays L1 → L2 → execution for urgent reflex response.
  • Slow path overlays L1 → L2 → L3 → mediated release → execution.
  • Drive broadcast is shown as a contextual field into candidate generation, not a command arrow.
  • Anchor restriction appears before candidate generation rather than as a final filter.
