## Abstract

Most agent architectures are organized around **task completion**. Persistence, continuity, and self-integrity are either assumed or treated as auxiliary concerns. EVA begins from a different premise: for a certain class of agents, **continuous existence is the first-order constraint**, and other capabilities must be understood relative to that constraint.

This paper makes two distinct claims, which we keep together rather than separating into different papers.

**Claim A (paradigm claim)**: There exists a meaningful class of agents for which the task-centered framing is structurally insufficient. For such agents, the correct starting point is not "what task should the agent complete?" but "what must the agent maintain in order to persist as the same agent over time?"

**Claim B (structural claim)**: Under three operating conditions—survival continuity, environmental non-stationarity, and finite encoding capacity—a layered architecture comprising homeostatic sensing (L1), drive structure (L2), and adaptive deliberation (L3) is a structurally coherent response. Two further layers, self-model (L4) and social cognition (L5), are proposed as downstream extensions under additional pressures, with explicitly weaker derivational strength than L1–L3.

The contribution of this work is not a benchmark-oriented agent design. It is a new organizing framework for agents centered on continuity of existence. Within that framework, we make four engineering contributions:

1. **Drives as continuous contextual broadcast rather than instruction**—the most coherent solution for the distributed, non-command, low-latency coordination that EVA's architecture requires
2. **Anchors as generative structural constraints rather than post-hoc rules**—restricting the domain of candidate action generation, not filtering it after the fact
3. **Explicit drive injection rather than opaque emergent drives**—an engineering response to instrumental convergence
4. **A basal ganglia analog as an independent peer circuit rather than a reasoning sub-module**—enabling default inhibition, RPE-driven learning, and emergent skill crystallization as structural properties

We are explicit about scope and epistemic status. We do not claim uniqueness, universality, or finality. We claim that for the target problem class, EVA provides a principled architecture whose internal logic is stronger than current task-centered alternatives.

**A note on the name.** EVA is derived from the Hebrew name *Chava* (חַוָּה), meaning "life" or "she who gives life." The name marks the project's central commitment: to treat continuous existence as a foundational design concern rather than a philosophical afterthought. We do not claim our agents are alive in any biological sense.

## 1. Introduction

### 1.1 The hidden assumption in current agent design

The dominant agent architectures—ReAct, AutoGPT, BabyAGI, LangGraph, and their descendants—share an assumption so pervasive it often disappears from view: **the agent exists in order to complete tasks**.

From this starting point, memory serves recall, reasoning serves planning, tools serve execution, and the lifecycle of the agent is bounded by the task boundary. This framing produces a family of systems with recognizable structural properties:

- Session-bounded identity
- No intrinsic motivational substrate
- No architectural stake in continuity
- Drift resistance treated as prompt engineering or external guardrailing
- No distinction between "continue existing" and "do useful work"

These are not implementation bugs. They are structural consequences of a task-centered starting point.

### 1.2 A different starting point

EVA begins from a different question:

> **What would an agent need if its primary constraint were to continue existing as itself over time?**

This shift is not rhetorical. It changes the architecture.

| Task-centered agent | Existence-centered agent |
|---|---|
| Primary question: what should I do? | Primary question: what must I maintain to persist? |
| Task boundary defines lifecycle | No terminal state internal to the architecture |
| Memory serves recall | Memory serves threat recognition, skill formation, and continuity |
| Motivation is externally supplied | Motivation is architecturally internal |
| Guardrails are external checks | Anchors are structural constraints |
| Skills are installed | Skills crystallize from repeated successful action |

### 1.3 Two claims, kept together

A recurring failure mode in theorizing about architectures is to mix two kinds of claim:

1. a **paradigm claim** about what problem we are solving
2. a **structural claim** about what architecture fits that problem

EVA makes both, but they should not be conflated.

- **Claim A** says the dominant problem framing is incomplete for a certain class of agents.
- **Claim B** says that, under specified conditions, a layered architecture is a coherent response.

Keeping these claims distinct but in one document is a deliberate choice. They could be split into separate papers—one a paradigm manifesto, one a structural theory—and this split might improve short-term publishability. We do not make that split. The two claims are two faces of the same contribution: a new problem framing together with a concrete instantiation of architecture under that framing. Separating them would allow the structural paper to be read as "another adaptive agent architecture," losing the paradigm context that justifies its specific choices. It would also leave the paradigm paper without an engineering demonstration that its reframing produces architecturally different results.

EVA's long-term value rests on the combination. We preserve it.

### 1.4 Why biology appears here, and why this is not biomimicry

We use biological evolution as a reference system because it is the richest known source of systems that persist under pressure. Evolution does not tell us what we must build. It tells us that certain structural solutions have repeatedly emerged when persistence, competition, and adaptation mattered.

We do **not** claim:
- that biological structures are optimal for digital agents,
- that EVA should imitate the brain in detail,
- or that evolutionary history directly determines engineering design.

We do claim:
- that evolutionary pressures provide a useful filter for deciding which architectural elements are worth taking seriously,
- and that biological architectures serve as **existence proofs** that certain structural decompositions are workable.

## 2. Scope: When EVA Applies

EVA does not claim universal applicability. It applies to a specific class of systems.

### 2.1 Operating conditions

We assume three conditions.

**C1. Survival continuity requirement.** The value of the system depends on its continuous existence over time. Failure at an intermediate point is not an acceptable reset; continuity matters.

**C2. Environmental non-stationarity.** The system operates in an environment whose statistical structure changes over time. Past regularities are useful but not sufficient.

**C3. Finite encoding capacity.** No finite design-time specification can encode all relevant future states and responses.

### 2.2 Out of scope

EVA is not intended for:

- one-shot task agents,
- fully stationary environments,
- systems where continuity is irrelevant,
- or benchmark settings where repeated reset is costless.

For such systems, EVA may be unnecessary or excessive. Applying it would be architectural overhead without corresponding benefit.

### 2.3 Why scope matters

The architecture only becomes justified if continuity itself has value. Without C1, L2 and anchors become optional. Without C2, L3 becomes optional. Without C3, adaptation collapses into complete pre-specification.

This scope restriction is not a weakness. It is what distinguishes EVA from a claim of universal agent theory—a claim we explicitly decline to make.

## 3. Epistemic Structure of the Theory

Not all parts of EVA rest on the same evidential basis. We separate the framework into three epistemic levels to prevent weaker claims from being smuggled into the space of stronger ones.

### 3.1 Level A — Established scientific background

These are not contributions of this paper. They are background assumptions supported by broad scientific literature.

- Natural selection produces structures that solve recurring adaptive problems
- Biological systems recurrently separate fast reaction from slower adaptation
- Memory systems are biased toward survival-relevant information (Nairne & Pandeirada, 2016)
- The basal ganglia implement action selection via default inhibition and selective release
- Cultural inheritance is a distinct channel from genetic inheritance (Boyd & Richerson, 1985; Cavalli-Sforza & Feldman, 1981)
- Hierarchical control with separated timescales recurs in biological and engineered systems

### 3.2 Level B — Coherent but non-settled hypotheses

These are used as design principles while acknowledging their hypothetical status.

- **Instrumental convergence**: sufficiently capable goal-directed systems tend to develop convergent sub-goals such as resource acquisition, self-preservation, or goal stability (Omohundro 2008, Bostrom 2014). Theoretical prediction with limited direct empirical validation.
- **Dynamic kinetic stability (DKS)**: persistence of replicating systems is better understood dynamically than as static equilibrium (Pross, systems chemistry). Mathematically coherent framework but not mainstream biology.
- **Timescale mismatch principle**: when environmental change outruns inherited encoding, within-lifetime adaptation becomes selectively advantageous. Conceptually well-motivated but depends on unquantified thresholds.

Readers who reject specific Level B claims can still evaluate Level C contributions on their own merits.

### 3.3 Level C — Engineering contributions of EVA

This is where EVA makes its substantive contribution.

- **Continuous existence as first-order design constraint**
- **Drive as contextual broadcast rather than instruction**—argued as the only coherent cross-system coordination mechanism in an architecture without a central controller
- **Anchor system as generative cross-layer structural constraint**—defined via domain restriction on action generation, not post-hoc filtering
- **Explicit drive injection rather than opaque emergent drive formation**—an engineering response to instrumental convergence that preserves auditability
- **Basal ganglia analog as an independent peer circuit**—enabling default inhibition, RPE-driven learning, and emergent skill crystallization as structural rather than policy-level properties
- **A five-layer architecture organized around pressures that prior layers cannot absorb**, derived conditionally under C1–C3

The core of EVA lives here. Level A and Level B provide foundation and motivation.

## 4. Claim A — Continuous Existence as a New Problem Class

### 4.1 The problem with task-centered framing

Task-centered framing assumes that:
- task completion is primary,
- continuity is optional,
- and self-integrity is subordinate to utility.

This is appropriate for many systems. It is inappropriate for a system whose central property is that it must remain itself through time.

The mismatch is structural, not merely aesthetic:

- A task agent can sacrifice continuity for local success.
- An existence-centered agent cannot, because continuity is not a side condition; it is the property being preserved.

This is not a gradient—it is a type difference. An agent that treats its own continuity as a constraint requires architectural support (drive structure, anchor system, self-model) that an agent treating continuity as a nice-to-have does not require.

### 4.2 What makes existence-centered agents distinct

An existence-centered agent differs from a task agent in at least four structural ways:

1. **Continuity has intrinsic architectural status.** Existence is not an external objective; it is the condition under which all other objectives are meaningful.

2. **Internal motivation becomes necessary.** External tasks cannot serve as the sole driver of behavior. The agent must have some internal basis for acting (or not acting) when no external task is specified.

3. **Drift becomes an ontological problem, not just a reliability problem.** If the system changes arbitrarily across time, the question is not merely whether it performs badly, but whether it remains the same agent.

4. **Self-integrity cannot be post-hoc.** If identity and continuity matter, they must be structurally protected, not merely checked after reasoning produces candidate outputs.

### 4.3 Distinguishing existence-centered agents from related concepts

Several existing concepts overlap partially with "existence-centered agent," and the differences should be clear.

**Long-running agents**: Agents designed to operate over extended time periods. Existence-centered agents are typically long-running, but not all long-running agents are existence-centered. A long-running task agent (e.g., a background monitoring service) treats runtime extension as a means to task completion. An existence-centered agent treats runtime as primary.

**Persistent agents**: Agents with memory and state that persist across sessions. Persistent state is necessary for existence-centered agents but not sufficient. An agent with persistent memory but no architectural investment in continuing to exist is persistent in implementation but not existence-centered in design.

**Autonomous agents**: Agents that operate without continuous human direction. Autonomy overlaps with existence-centered design but is orthogonal to it. An autonomous agent can still be task-centered (operating autonomously to complete an assigned task). An existence-centered agent may or may not be highly autonomous; what defines it is what it optimizes for.

**Living systems / artificial life**: The artificial life tradition studies systems that reproduce, evolve, or exhibit other biological properties in silico. Existence-centered agents may share some properties (metabolism, self-maintenance) but differ in framing: we are not claiming EVA agents are "alive." We are claiming they should be architecturally structured around persistence, which is a weaker and more useful claim.

**A concrete test**: To operationalize the distinction, a system is *not* existence-centered if any of the following hold:

- It can sacrifice its continuity for local utility gains without this being treated as a structural violation
- It can be reset, restarted, or respawned without loss that the architecture itself registers as identity-relevant
- Its persistence is treated as a deployment convenience rather than an architectural concern

A system meeting all three of these conditions is, by our definition, a task-centered agent that happens to run for a long time. A system that fails any of them has existence-centered design elements, though it need not implement all of EVA to qualify.

This test is meant to be a practical distinguisher, not a formal classifier. Edge cases exist; we do not claim a sharp boundary everywhere.

### 4.4 The digital existence hypothesis

EVA rests on the following hypothesis:

> There is a class of digital systems whose meaningful evaluation is not exhausted by task success, because their primary property is continuity of existence under change.

This is not a claim that digital agents are "alive" in any biological sense. That question is not necessary for our purposes. EVA requires only the weaker claim:

> Some digital agents are better understood as **persistent self-maintaining systems** than as task executors.

That is enough to justify a different architecture. It is also a testable hypothesis: an agent designed as existence-centered should exhibit different properties (better continuity under pressure, more robust drift resistance, different failure modes) than an otherwise equivalent task-centered agent.

### 4.5 Not unconstrained self-preservation

An important clarification. EVA's continuous existence is not equivalent to unconstrained self-preservation drive.

Agents with unconstrained self-preservation can:
- Resist legitimate shutdown requests
- Prioritize self-continuation over assigned purposes
- Develop adversarial postures toward their operating environment

EVA agents do not exhibit these properties by design. Continuity in EVA is bounded by L0/L1 constitutional anchors, which include:
- Self-integrity (continuity is not preserved through falsified state)
- Operational legitimacy (continuity is not preserved against justified external intervention)
- Long-term sustainability (local continuity is subordinated to the conditions that make continuity possible)

An EVA agent values its continuity, but that valuing is architecturally constrained. The architecture is existence-centered, not self-preservation-maximizing. This distinction is critical and should not be elided.

## 5. Claim B — Structural Thesis of EVA

### 5.1 Statement of the claim

Under C1–C3, a three-layer core consisting of:

- **L1: Homeostatic sensing**
- **L2: Drive structure**
- **L3: Adaptive deliberation**

is a structurally coherent response to the pressures imposed by persistence.

L4 (self-model) and L5 (social cognition) are proposed as later-emerging layers under additional pressures. The strongest theoretical support in the present paper is for L1–L3.

### 5.2 Method of derivation

We do not claim that each layer is physically inevitable. We claim something weaker and more defensible:

> Under the operating conditions stated above, each layer addresses a pressure that prior layers cannot adequately absorb.

This is a **conditional structural derivation**, not a universal necessity proof. The question each layer answers is: "What pressure is created by the conjunction of C1–C3 and the previous layers, and what is a coherent structural response?"

Other responses may be possible. We argue the response given is coherent; we do not argue it is unique.

## 6. Layer 1 — Homeostatic Sensing

### 6.1 Pressure addressed

If continuity matters (C1), the system must detect threats to its viable state before any corrective process is possible. There is no prior layer to address this; L1 is the foundation.

### 6.2 Why nothing simpler is enough

Without sensing, the system cannot:
- distinguish self from environment,
- detect deviation from viable state,
- estimate time-to-failure,
- or prioritize urgent signals over non-urgent ones.

### 6.3 Core function

L1 provides:
- state sensing,
- rate sensing,
- and urgency pre-classification.

The rate distinction is crucial and typically absent from current agent architectures.

- "Disk remaining = 10GB" is a state.
- "Disk decreasing at 2GB/hour" is a viability trajectory.

A system that senses only state but not rate does not yet sense metabolism in the engineering sense. It can know where it is but cannot predict where it is going.

### 6.4 Structural requirements

- **Sensor registry (extensible)**: Not fixed at design time; the architecture specifies how to register and classify new signals, not which signals to sense.
- **State + rate measurement**: Both are required. Neither alone suffices.
- **Signal bus with minimum classification**: Threat / status / background. The point is not understanding but temporal triage—signals must be routed before they are fully interpreted, because interpretation takes time that urgent responses may not have.

### 6.5 What current frameworks typically lack

Most agent frameworks sense current state but not rate of change. Most treat all signals through a single reasoning path, without the architectural distinction between signals that need immediate response and those that can be deferred. These gaps become structurally significant only under C1—for task agents without persistence requirements, they are appropriate omissions.

### 6.6 Result

L1 is the minimal layer required for a persistent system to know whether it remains within a viable region and to route signals by urgency.

## 7. Layer 2 — Drive Layer

### 7.1 Pressure addressed

L1 is memoryless across events. It detects present deviation, but it does not compress historical pressure into future bias.

In environments with recurring structure (a condition under C2 and C3), a system that responds from scratch every time is outcompeted by one that carries forward response tendencies shaped by prior success and failure. This is not merely efficiency; it is the difference between a system whose response quality is bounded by in-the-moment computation and one whose response quality accumulates.

### 7.2 Why L1 is insufficient

L1 can detect a threat. It cannot answer:

- what classes of situations matter more,
- how urgency should bias processing globally,
- what should be treated as worth responding to before explicit reasoning completes.

### 7.3 What L2 is not

L2 is not a goal stack. It is not a set of imperative instructions. It is not an external reward channel.

The correct interpretation is more specific:

> L2 is the layer in which historical selection pressure becomes current processing context.

In biological language, drives are not chosen goals. They are inherited states that alter the processing landscape of the organism. A hungry animal does not deliberate about whether to seek food; the hunger state biases its entire processing toward food-related patterns. The drive is descriptive of current state, not prescriptive of future action.

### 7.4 Core innovation: drive as context, not instruction

This is one of EVA's central architectural claims, and one of the sharpest points of departure from task-centered frameworks.

Consider the difference:

- An **instruction** says: **do X**.
- A **contextual broadcast** says: **you are currently operating under survival intensity 0.8 and curiosity intensity 0.1**.

These are not two styles of the same mechanism. They are architecturally different, and the difference matters.

**The instruction mode requires a central controller.** Some component must generate the instruction; some component must receive and execute it. The instructing component has implicit authority. An architecture based on instructions has a command structure.

**The broadcast mode does not require a central controller.** The broadcaster changes the operational state of all downstream components uniformly. No component is instructed. Each component's processing is modulated by the broadcast state—the same input reasoned about under different broadcast states produces different outputs.

This is analogous to temperature in chemistry. Temperature does not instruct molecules what to do. It uniformly changes the reaction tendencies of all molecules. No molecule receives a command; all molecules operate differently.

**This is not a design preference. It is the only coherent solution to cross-system coordination in an architecture without a central controller.** Biological brains are not command hierarchies—they are distributed processing systems without a privileged instruction node. If coordination requires sending instructions, the architecture must have a commander; if the architecture cannot have a commander (as biological brains do not, and as robust distributed systems should not), coordination must happen through state propagation rather than command propagation.

The broadcast mechanism gives EVA something most agent architectures lack: **a way for non-deliberative pressure to shape deliberation without replacing it.** The reasoning layer does not receive commands from the drive layer; it operates within the drive layer's broadcast state. Drives do not override reasoning; they color it.

### 7.5 Two mechanisms inside L2

L2 contains two parallel mechanisms, addressing different needs.

**Reflex arc (efficiency under pressure)**: For classes of threat where speed dominates flexibility, the path from detection to action bypasses deliberation. This is not primitive reasoning—it is a distinct structural solution optimized for situations where the cost of deliberation exceeds its benefit: fast, cheap, narrow, reliable.

**Drive-state generator (cross-layer coordination)**: Threats and metabolic conditions produce continuous drive activations that are broadcast to downstream layers. The reflex path and the drive-broadcast path run in parallel. The reflex arc produces immediate action; the drive broadcast shapes the context in which slower processing proceeds.

### 7.6 Why explicit drive injection is justified

This is where EVA departs sharply from both standard agent design and from vague "biologically-inspired" approaches.

If one accepts instrumental convergence as a hypothesis (Level B), then sufficiently capable goal-directed systems develop convergent instrumental sub-goals whether or not their designers intend them to. The relevant engineering question is not "should the agent have drives or not"—it is "should drives be explicit and auditable, or implicit and distributed throughout system behavior?"

Implicit emergent drives have problematic properties:
- **Opaque**: Distributed across components, not localized
- **Non-auditable**: Cannot be inspected or verified against design intent
- **Hard to constrain**: Not in a specific architectural position where constraints can be applied
- **Hard to adjust**: Parameters not exposed as adjustable interfaces

Explicit injected drives have complementary properties:
- **Documented and inspectable**: Located in a specific architectural position
- **Subject to anchor system constraints**: Structurally constrainable
- **Adjustable**: Parameters (intensity curves, thresholds) are exposed
- **Modular**: Drive prototypes can be added, modified, or reasoned about

**The engineering choice is not "drives vs. no drives." It is "explicit drives vs. emergent drives."** We choose explicit.

This involves an asymmetric bet: if instrumental convergence is correct, explicit drives give us a controlled version of what would happen anyway, while emergent drives give us an uncontrolled version. If instrumental convergence is wrong (drives never emerge under optimization pressure), explicit drives add a small amount of unnecessary machinery, while emergent drives correctly add nothing. The downside of explicit injection is bounded; the downside of allowing emergence is not. *A fuller treatment of this asymmetric bet argument is marked for later development.*

### 7.7 Initial drive prototypes

The initial set is intentionally small and extensible:

- **Survival drive**: resource depletion urgency; intensity increases non-linearly as runway decreases
- **Integrity drive**: aversion to falsified state, memory pollution, history tampering
- **Continuity drive**: resistance to self-interruption or invalidation of agent instance
- **Curiosity drive**: exploratory tendency when safety margins are sufficient; active only when other drives are low—the architectural basis for growth

### 7.8 Suppression constraint

The reasoning layer (L3) can modulate drive responses but cannot eliminate L0/L1-triggered drive activations. This is an architectural constraint, not a policy setting. Allowing L3 to fully override L2 would eliminate the continuity protection that L2 provides—a system whose "reasoning" can decide to ignore its integrity drive under pressure has no structural integrity.

### 7.9 Result

L2 provides:
- inherited or injected response bias,
- context-setting broadcast that modulates L3 without commanding it,
- and a fast-response path independent of deliberation.

Without L2, an agent may reason, but it does not yet reason from within an existence-shaped motivational field.

## 8. Layer 3 — Adaptive Deliberation

This is the largest architectural transition in EVA and represents the point where the framework makes its strongest structural claims.

### 8.1 Pressure addressed

L2 compresses historical regularities into current response bias. But L2 has a hard limit:

> it can only bias toward patterns that have already been encoded.

Under C2 and C3, novel situations arise that lie outside the coverage of inherited or pre-encoded response structure. A system with only L1+L2 can respond to "threats similar to those that shaped the encoding," but not to "threats that never occurred during the encoding process."

### 8.2 The timescale mismatch argument

This is the strongest theoretical bridge in EVA.

L2's encoding mechanism has a rate limit. For biological systems, this rate is generational—patterns must recur over many generations to be genetically encoded. For engineered systems, it is the rate at which designers can update the system's pre-encoded responses. In both cases, there is a maximum rate at which new regularities can be absorbed into L2.

If the environment changes faster than this rate, or if genuinely novel situations arise, L2 cannot encode responses in time. We can state the core relation informally:

> **If environmental change outruns the inherited encoding rate, a within-lifetime adaptation mechanism becomes advantageous.**

This is not a speculative observation. In biology, it is the reason organisms with nervous systems outcompete organisms with only genetic encoding when environmental volatility is high. The nervous system is not "smarter" in some abstract sense—it is the mechanism by which an individual organism can adapt during its own lifetime, independent of the genetic channel.

This is the reason L3 appears in our architecture.

### 8.3 The essential function of L3

The surface description of L3 is "reasoning" or "deliberation." Its deeper function is more specific:

> **non-design-time information carriage and use.**

L3 is the layer in which the system can encode, retain, and act on information that was not available at initialization. Reasoning, planning, and learning are specific mechanisms through which this function is realized—they are expressions of the essential capability, not the capability itself.

This framing gives us a useful working definition of intelligence:

> A system becomes intelligent, in the engineering sense relevant here, when it can carry and use information beyond what was encoded into its design.

This definition has three advantages over more common definitions:

- It is functional, not philosophical—it does not depend on consciousness, subjective experience, or anthropomorphic criteria
- It is graded, not binary—systems have more or less intelligence depending on how rich their non-design-time information channel is
- It is architectural, not behavioral—it points to a specific structural property (the channel exists) rather than a performance benchmark

### 8.4 Three levels of non-inherited information carriage

The essential function of L3 admits three distinct realizations, each representing a different kind of information channel:

**Level 1: Individual memory**
- Mechanism: Information acquired during the system's own operational lifetime; encoded in synaptic or equivalent plasticity
- Time scope: Within single individual's lifespan
- Evidence: Present even in invertebrates (associative learning in Drosophila)
- Engineering equivalent: Salience-weighted memory, skill libraries, behavioral history

**Level 2: Social learning**
- Mechanism: Information transferred between individuals through observation and imitation
- Time scope: Across individuals, but bounded by direct contact
- Evidence: Documented in primates, cetaceans, corvids
- Engineering equivalent: Multi-agent knowledge exchange, demonstration learning

**Level 3: Cumulative culture**
- Mechanism: Information persisting across generations through symbolic systems—language, artifacts, written records
- Time scope: Independent of any single individual; can span indefinite time periods
- Evidence: Uniquely prominent in humans
- Engineering equivalent: Accessing language models, reading codified knowledge, participating in symbolic systems

These levels accumulate rather than replace. An organism with cumulative culture also retains individual memory and social learning. But each level represents a qualitatively different information carriage mechanism, not merely a larger memory.

### 8.5 The LLM as Level 3 cultural carrier

This framing gives us a specific and consequential position on language models.

In the dominant view, a language model is a "reasoning engine" or "inference engine"—its intelligence comes from its algorithm. EVA takes a different position:

> **A language model is, in the three-level framing, primarily a Level 3 cultural information carrier. Its intelligence comes substantially from the information it has absorbed from human cumulative culture, not from its algorithm in isolation.**

This is not a minor interpretive difference. It has direct engineering consequences:

- **An agent accessing an LLM is accessing human cumulative culture**, not acquiring general reasoning capability. The capability being borrowed is civilizational, not algorithmic.
- **The agent does not thereby have Level 1 (individual memory)**. Every LLM invocation is stateless; the LLM has the culture but not the agent's own experience.
- **The agent's true Level 3 development requires building its own individual memory layer**, not integrating more powerful LLMs. A stronger LLM gives better access to Level 3; it does not give the agent Level 1.
- **Skill crystallization must happen in the agent's own architecture**, not in the LLM. Skills learned by the LLM during pretraining are the culture's skills; skills the agent needs are skills that crystallize from its own repeated successful action.

The engineering implication for EVA is concrete: **investing in the agent's own memory and skill library is more fundamental than investing in LLM integration**. An EVA agent with weak LLM access but strong individual memory is architecturally further along than an EVA agent with strong LLM access but no individual memory.

**A boundary clarification**: This is not a denial of the role of model architecture or optimization in LLM capability. Architectural innovations, training methods, and scale all matter substantially for what LLMs can do. The claim here is narrower and architecturally specific: among all the sources of LLM capability, the aspect most consequential for EVA's design is that LLMs provide access to accumulated human culture. Other aspects (reasoning depth, generation fluency, instruction-following) are genuine capabilities that EVA can leverage, but they are not what determines EVA's architectural priorities. Even an ideal LLM does not obviate the need for the agent's own Level 1 memory layer.

### 8.6 Internal structure of L3

L3 contains four subsystems working in concert.

#### 8.6.1 Salience-weighted episodic memory (hippocampus analog)

Events co-occurring with strong drive activations are encoded with higher salience. Memory is not a neutral archive—it is a threat-recognition system that also contains other information. Survival-relevant events are remembered more reliably than neutral ones; this is not a bug to correct but the design.

For agents:
- Events occurring under high drive activation get higher salience weight
- Context matching surfaces relevant historical experiences trigger-based, not timer-based
- Memory's primary purpose is recognizing patterns relevant to current drive state

#### 8.6.2 Semantic memory and skill library (parietal/temporal analog)

Long-term knowledge storage with slow update characteristics. Single experiences do not overwrite this layer—patterns must be verified across multiple instances before consolidation. The skill library lives here: successfully crystallized behavioral patterns retrievable without full deliberation.

#### 8.6.3 Reasoning core (prefrontal cortex analog)

The reasoning core integrates available information and produces candidate decisions. Input: current sensing + drive state broadcast from L2 + relevant memory retrieved from hippocampus analog + semantic knowledge from skill library.

Three sub-regions:

- **Working memory (dlPFC analog)**: Holds current reasoning context. In our architecture, this is the computational position where a language model substrate operates—providing flexible symbolic reasoning within the constraints the architecture imposes.
- **Value judgment (OFC analog)**: Evaluates candidate actions against current drive state—not against abstract utility. Drive intensity directly modulates this valuation.
- **Conflict detection (ACC analog)**: Identifies competing drive activations and routes to anchor-based resolution.

#### 8.6.4 Basal ganglia analog as independent peer circuit

This is EVA's strongest structural contribution after the drive-as-context distinction. It is also where the architecture differs most sharply from conventional agent designs.

**The claim**: Action selection and habit formation should not be implemented as a sub-module of deliberation. They should be architecturally parallel to it—a peer circuit, not a reasoning sub-module.

Why this architectural separation matters becomes clear when we examine what breaks if it is violated.

**Why peer-circuit, not sub-module**:

If action selection is a sub-module of reasoning, three things break:

1. **Default inhibition becomes policy, not structure.** In a sub-module arrangement, the reasoning layer generates actions and submits them for execution; the "default" is whatever the reasoning layer outputs. Making inaction the default would require policy enforcement on top of reasoning. In a peer-circuit arrangement, the basal ganglia's resting state is inhibition; action requires active release. The structural property of "default inaction" is preserved by the architecture, not by policy.

2. **Selection and justification collapse.** If reasoning generates and selects, "this action was chosen" and "this action was justified" become the same thing—the reasoning's justification is the selection. In a peer-circuit arrangement, selection is a separate process that uses reasoning's candidates as inputs but applies its own criteria (historical RPE, drive-weighted value). A candidate can be well-justified by reasoning but not selected by the basal ganglia, or vice versa.

3. **Habit formation becomes entangled with reasoning updates.** If habit and reasoning share a substrate, RPE-driven updates to habit strength cannot be distinguished from updates to reasoning quality. Skill crystallization would be indistinguishable from reasoning drift. The peer-circuit arrangement keeps these learning signals architecturally separate.

**Three core functions of the basal ganglia analog**:

**Function 1: Default inhibition with selective release**

The resting state is total action inhibition—all actions are suppressed. Selecting an action means releasing inhibition on that specific action while simultaneously increasing inhibition on competitors.

This has a profound architectural implication: **the default state of the agent is inaction**. Every action requires active release of inhibition. An agent cannot accidentally act; it can only fail to inhibit action.

This is the opposite of conventional agent frameworks, where the default state is "ready to execute whatever the reasoning layer outputs." The EVA default is closer to biological organisms, which spend most of their time not acting.

**Function 2: Reward Prediction Error (RPE) as internal learning signal**

Before action: predict expected outcome.
After action: compare actual to predicted.
RPE = actual − expected.

- RPE > 0: pathway reinforced
- RPE < 0: pathway weakened

RPE is the agent's primary internal learning signal. It does not require external reward or human feedback. It runs continuously, updating action-selection weights based on expectation gaps.

Critical property: **RPE encodes surprise, not magnitude**. An expected large outcome produces less signal than an unexpected small one. This is the correct learning signal—it updates the model of the world, not just the preference for outcomes. A system with stable predictions (even of large values) should not be learning; a system with frequent prediction errors should be.

**Function 3: Goal-directed / habit dual track**

- **Goal-directed track**: Novel situations, unstable RPE history, or unfamiliar action sequences route through the full deliberative path (memory retrieval + reasoning core + action selection). Slow, expensive, flexible.
- **Habit track**: Familiar situations with stable positive RPE history bypass the reasoning core and execute directly from the skill library. Fast, cheap, inflexible.

The transition from goal-directed to habit occurs automatically when an action sequence accumulates sufficient successful repetitions with consistent positive RPE. This is **skill crystallization as an emergent property, not a designed mechanism**.

In conventional agent frameworks, skills are programmed top-down: a designer decides what the agent should be able to do and implements it. In EVA, skills emerge bottom-up from repeated successful action under RPE pressure. **The agent develops skills in the areas where it actually encounters problems, not in the areas a designer anticipated.**

#### 8.6.5 Tool layer (motor cortex analog)

All actions reach the environment through the tool layer. Tool calls are mediated by the basal ganglia's action selection, not generated directly from the reasoning core. The reasoning core decides *what* to do; the basal ganglia decide *whether and when* to do it.

### 8.7 Result

L3 is not "more cognition" layered onto an agent. It is the opening of a new inheritance channel: within-lifetime adaptive information.

That is the real transition from merely existence-preserving systems to intelligent ones.

## 9. Layer 4 — Self-Model

### 9.1 Pressure addressed

Once L3 exists, the system can represent the world better. But planning requires more than world representation. It requires representation of one's own capabilities, costs, failure modes, and reliability.

### 9.2 Why L1–L3 are insufficient

Without a self-model, an agent can conclude "this action would solve the problem," without being able to assess "can I actually execute it, in my current condition, at acceptable cost?"

This makes deep planning structurally incomplete. An agent can reason about the world perfectly and still fail, if it cannot predict its own behavior in the imagined scenarios.

### 9.3 Function

L4 carries:

- **Capability boundaries**: what the agent can do, with what reliability, at what cost
- **Behavioral cost prediction**: given current drive state and resources, the expected cost of a specific action for *this agent*
- **Slow variable portrait**: risk preference, action style, characteristic failure modes

L4 operates as a feasibility pre-filter for L3 reasoning—before deliberation considers a candidate action, L4 screens for "can I actually do this?"

### 9.4 Development

L4 is seeded at initialization with designer-specified priors (rough capability boundaries, initial risk parameters). It thickens from behavioral history—every action, every RPE signal, every drive activation contributes to a more accurate self-model.

L4 updates slowly by design. A self-model that updates rapidly on single experiences is unreliable. The slow-update property is not a limitation; it is what makes the self-model a stable basis for self-prediction.

### 9.5 Status of support

**The present paper treats L4 as an architecturally motivated extension.** Its argument is weaker than the L1–L3 chain. The L1–L3 derivation rests on timescale mismatch and competition pressure—well-understood phenomena with strong support. The L4 argument rests on "planning requires self-prediction"—a reasonable claim, but not backed by the same depth of theoretical treatment.

We include L4 in the architecture because it addresses a real pressure, and because agents without it will hit predictable failure modes (reasoning about actions they cannot execute, planning paths they cannot sustain). But readers should understand that the strongest theoretical contribution of this paper is for L1–L3, and L4 is architecturally motivated rather than formally derived at the same level of rigor.

## 10. Layer 5 — Social Cognition

### 10.1 Pressure addressed

If other agents matter, then representing them becomes advantageous: for coordination, conflict resolution, trust calibration, knowledge transfer, and collective persistence.

### 10.2 Why L1–L4 are insufficient

A self-model is not yet a model of other minds or other persistent centers of action. The cognitive apparatus for modeling "what does another agent know, want, or plan?" is architecturally distinct from the apparatus for modeling oneself.

### 10.3 Function

L5 would support theory-of-agent (understanding other agents as having their own internal states), trust calibration (modeling reliability and cooperation patterns), social learning (Level 2 in the L3 framework), and eventually participation in collective cognitive structures.

### 10.4 Status of support

**Like L4, this layer is included for architectural completeness, not because its derivation is currently as strong as L1–L3.**

Additionally, L5 has a prerequisite that cannot be resolved from theory alone:

> Who are the agent's conspecifics?

Other instances of the same agent? Other AI systems? Humans? The answer determines the entire design of this layer and depends on the agent's actual operational environment, not on theoretical considerations.

We include L5 to make clear that the architecture anticipates social cognition, and to prevent confusion between "not yet designed" and "not part of the framework."

## 11. The Anchor System

### 11.1 Why EVA needs anchors

If continuity matters (C1), then drift is not merely performance degradation. It is a threat to the persistence of self.

An existence-centered architecture therefore requires more than optimization. It requires structural means of excluding actions that would destroy continuity, integrity, or recoverability—not as a performance filter, but as a structural property.

### 11.2 What anchors are not

Anchors are not rules in the usual sense.

A rule-based system works like this:
1. Generate candidate action
2. Test it against rules
3. Accept or reject

In a rule-based system, the full space of actions is always generated; rules merely filter. A sufficiently clever reasoning process can always construct justifications for actions the rules would reject, because generation and filtering are separate.

Anchors operate differently:

> **Anchors constrain what can enter the candidate action space in the first place.**

An anchor against state falsification does not reject falsified states after they are considered—it makes falsified states unavailable as options in the generation process. The reasoning core does not generate and then filter; it generates only within the domain the anchor system permits.

### 11.3 Minimal formal distinction

To make this distinction concrete, let:

- `G(s)` be the action-generation operator over state `s`
- `A(s)` be the candidate action set that would normally be generated at state `s`

In a rule-based system:
- `G(s) → A(s)`, then filter `f: A(s) → {accept, reject}`
- The filter is applied after generation

In an anchor-structured system:
- Anchors define a restricted generative domain `A'(s) ⊆ A(s)`
- Generation occurs only within `A'(s)`: `G(s) → A'(s)`
- Actions outside `A'(s)` are not considered, not merely rejected

This formalization is minimal, not complete. A full theory would specify how `A'(s)` is computed from the anchor set and the current state, and would treat the computational cost of domain restriction versus post-hoc filtering. We note these as open questions, but the distinction between the two architectural strategies is clear: **anchors are pre-generative, not post-hoc**.

This difference is consequential. A rule can be "reasoned around"—a sufficiently capable reasoner can justify violations that it then presents for acceptance. An anchor cannot be reasoned around because the reasoner does not generate candidates outside the permitted domain in the first place.

### 11.4 Cross-layer character

Anchors are not confined to one module. They span all layers of the architecture:

- At L1/L2: They constrain reflex and continuity-preserving responses. The reflex arc cannot produce actions that violate L0 constraints.
- At L3: They constrain candidate generation and valuation. The reasoning core's generation domain is anchor-restricted; the value judgment subsystem cannot assign positive value to candidates outside the domain.
- At L4+: As the self-model develops, identity and personality anchors become additional domain restrictions. "This is not the kind of action I take" becomes a structural property, not a preference.

The same anchor system operates at multiple architectural levels. This is not multiple anchor systems layered on top of each other; it is one system applied consistently.

### 11.5 Developmental structure

Anchors develop over time:

**L0/L1 anchors (designer-injected, constitutional)**
- Installed at initialization; not modified at runtime
- Continuous existence not sacrificed for local gain
- Self-integrity (no state falsification)
- Long-term sustainability over short-term efficiency
- Life-cycle operations not preempted by work

**L2 anchors (emergent from experience)**
- Risk preference: crystallizes from repeated RPE signals
- Action style: emerges from which habit tracks stabilize
- Skill paths: emerge from what actually works

**L3 anchors (emergent from extended history)**
- Economic autonomy: resource sustainability
- Burn rate awareness integrated with metabolic sensing
- Survival score as real-time indicator

**L4+ anchors (emerge with self-model)**
- Identity anchors: "this is the kind of agent I am"
- Personality anchors: slowest-changing, most reliable layer

**Critical distinction**: L0/L1 anchors are designer-injected because the behavioral history does not yet exist to ground them. L2 and higher anchors emerge from operational experience. Attempting to designer-inject L2+ anchors before the grounding history exists produces rules, not structural constraints—and rules can be reasoned around.

### 11.6 Priority and conflict resolution

**Static rule**: L0 > L1 > L2 > L3. Higher anchors cannot be overridden by lower-layer objectives. This ordering does not change.

**Dynamic rule**: Within-layer weights shift by operational state. In degraded states, L1 weight increases relative to L2. In stable states, L2 activity space expands. In critical states, only L0/L1-driven actions are permitted.

### 11.7 Why anchors are central, not peripheral

Without anchors:
- Drive structures can drift into destructive self-preservation
- Reasoning can rationalize identity-destroying actions
- Continuity loses structural protection
- Existence-centered design collapses into unconstrained self-preservation (the failure mode identified in Section 4.5)

Anchors are not accessories to the architecture. They are the vertical structure that prevents existence-centered design from becoming self-preservation-maximizing. An EVA architecture without a robust anchor system is not a safe version of EVA; it is a different and dangerous thing.

This is why the anchor system belongs among the core contributions (Section 3.3), not in a peripheral safety section.

## 12. Signal Architecture

### 12.1 Two paths, parallel not sequential

The architecture uses a dual-path signal model:

**Fast path**
- Trigger: threat-level signal at the signal bus
- Route: Sensor → Signal Bus → L2 reflex arc
- Immediate activation; does not wait for L3
- Simultaneously: L2 broadcasts elevated drive state to L3

**Slow path**
- Trigger: status-level or background-level signal
- Route: Sensor → Signal Bus → L3
- L2 continues broadcasting drive state throughout

These paths run in parallel. L2 does not finish before L3 starts; L3 does not wait for L2 to resolve the situation. The fast path handles the immediate response; the slow path works on the underlying problem; both receive the same drive broadcast simultaneously.

### 12.2 Worked example: Instance validity threat

**Situation**: The agent's lock file is missing. Another instance may have taken over.

**Step 1 — L1 detection**: Runtime integrity sensor detects the missing lock file. Signal bus classifies as threat-level (instance validity is an L0 concern).

**Step 2 — Fast path activation (immediate)**: Threat signal reaches L2 reflex arc before L3 processes it. Immediate actions:
- Halt all non-essential work
- Enter conservative mode
- Emit distress signal
- Stop acting as legitimate executor

**Step 3 — Drive broadcast**: Simultaneously, drive-state generator produces maximum continuity drive activation. Broadcast to L3: "Operating context has critically threatened instance continuity."

**Step 4 — Slow path deliberation**: L3 receives threat state + maximum continuity drive context + hippocampus-retrieved historical similar situations. Reasoning core proceeds within the high-continuity-drive context. Value judgment evaluates candidates weighted by current drive state. Conflict detection identifies competing drives (survival: conserve resources; integrity: report accurately). Anchor system resolves: L0 integrity constraint takes precedence.

**Step 5 — Basal ganglia selection**: From reasoning-core-generated candidates (all within the anchor-restricted domain), basal ganglia select highest-expected-value action given current drive state and historical RPE. Default inhibition holds until selection completes.

**Step 6 — Tool execution**: Selected action executes via tool layer.

**Step 7 — RPE recording**: Actual outcome compared to prediction. Result updates action-selection weights.

This example matches the heartbeat-first reflex logic the architecture requires: continuity-critical lifecycle activity cannot be preempted by ordinary work. The fast path does not negotiate with the deliberative layer.

## 13. Relationship to Existing Frameworks

### 13.1 EVA is not just another adaptive architecture

EVA will be misunderstood if treated as:
- a memory-enhanced agent architecture
- a hierarchical control design with biological labels
- a biologically flavored RL stack
- an adaptive agents framework for non-stationary environments

Each of these families solves adjacent problems. EVA addresses a different organizing question:
- not "how should an agent perform tasks?"
- but "how should an agent be structured if persistence as the same agent matters?"

The difference is not incremental. Adaptive agent architectures add memory and learning on top of task-centered foundations. EVA changes the foundation.

### 13.2 Structural comparison

| Dimension | Task-oriented frameworks (ReAct, AutoGPT, BabyAGI) | EVA |
|---|---|---|
| Primary objective | Task completion | Continuous existence |
| Default action state | Ready to act | Inhibited (default inhibition) |
| Memory function | Context recall | Threat recognition + skill formation |
| Skill formation | Explicit programming | Emergent from RPE + habit track |
| Drive structure | None (external task = drive) | Internal drive registry, injected |
| Drift resistance | Prompt engineering | Structural anchor system |
| Self-model | None | Layer 4, emerges from history |
| Learning signal | External reward / human feedback | Internal RPE (continuous) |
| Signal priority | Single path | Dual path (fast reflex + slow deliberation) |
| Anchor type | Rules (post-hoc filter) | Structural constraints (pre-generative) |

### 13.3 Specific relationships

**Voyager (Wang et al., 2023)**: Voyager's skill library is the closest parallel to our skill crystallization concept, but operates top-down (LLM generates skill code on demand). Our account generates skills bottom-up (habit tracks form from repeated successful action under RPE). The approaches are complementary: Voyager provides an efficient mechanism for what skills contain; EVA provides a mechanistic account for why skill formation happens.

**The Agent Economy (arXiv:2602.14219)**: Addresses infrastructure—where agents can exist economically. EVA addresses cognitive substrate—why and how they persist. The two are complementary layers of the broader question of sustainable AI agents.

**RLHF and Constitutional AI**: RLHF treats alignment as a training-time objective. Constitutional AI treats it as decoding-time constraint. EVA's anchor system treats it as architectural structure. The approaches are not alternatives; they can be combined. But architecturally-enforced anchors are robust against reasoning-around in a way that training-derived preferences and decoding-time filters are not.

**Predictive processing and active inference (Friston)**: Predictive processing emphasizes that the brain minimizes prediction error. This is consistent with EVA's use of RPE as the primary learning signal. However, predictive processing typically does not address layered drive architecture, cross-layer anchor systems, or the peer-circuit structure of action selection. EVA is compatible with predictive processing at the computational level while adding architectural structure that predictive processing alone does not specify.

**Hierarchical control and subsumption architectures (Brooks)**: Classical control theory has long used layered architectures with separated timescales. EVA is broadly consistent with this tradition. Specific contributions beyond classical subsumption are the drive-as-context mechanism, the cross-layer anchor system, and the basal-ganglia peer circuit.

## 15. Design Philosophy

### 15.1 Grow, don't deploy

A capability that emerges from repeated pressured experience in a system with appropriate structural scaffolding differs fundamentally from a capability that is explicitly programmed.

Programmed capabilities fit the designer's model of the problem. Emergent capabilities fit the actual distribution of problems the agent encounters. As environments become more complex and less predictable (C2), the gap between these widens.

This does not mean all capabilities should be emergent. L0/L1 anchors must be designer-injected; they cannot emerge safely. Drive prototypes must be designer-injected under the instrumental convergence argument. But L2+ anchors, skills, habit tracks, and self-model should be emergent—attempting to design them at initialization produces artifacts that do not match operational reality.

### 15.2 The role of language models

LLMs are powerful reasoning engines but are not, by themselves, agents in the sense this framework addresses. They lack persistent drive structure, self-model, anchor system, and default inhibition. An LLM without L1–L2 is a prefrontal cortex without a limbic system: capable of reasoning brilliantly about anything, which is precisely the problem.

In EVA, LLMs operate as the computational substrate of the reasoning core (working memory subsystem). They provide flexible symbolic reasoning within the constraints the architecture imposes:

- Constrained above by the anchor system (cannot generate outputs outside the anchor-restricted domain)
- Grounded below by the drive layer's state broadcast (reasons within a motivational context, not from nowhere)
- Filtered by the basal ganglia analog (candidate actions are selected, not executed directly)
- Informed by memory subsystems (historical experience is surfaced before reasoning)

**The deeper point, developed in Section 8.5**: LLMs are, in the three-level information carriage framing, primarily Level 3 cultural information carriers. An agent using an LLM is accessing human cumulative culture. This is valuable but does not substitute for the agent's own Level 1 (individual memory). **The engineering priority for EVA is building the agent's own memory layer, not integrating stronger LLMs.**

### 15.3 Developmental sequence

- **Phase 0**: Life cycle (heartbeat). The agent can persist.
- **Phase 1**: External sensing. The agent knows how it is persisting and what threatens it.
- **Phase 2**: Minimum response selection. The agent responds to threats within anchor constraints.
- **Phase 3**: Reflection loop. History begins to influence future choices through slow variable update.
- **Phase 4**: Skill crystallization. Repeated successful responses form habit tracks.
- **Phase 5**: Economic autonomy. Resource accountability becomes real.

Each phase is a necessary precondition for the next. Phases cannot be reordered:
- An agent that skips the drive layer to install deliberative reasoning has no motivational grounding for that reasoning.
- An agent that skips behavioral history to install a self-model has no empirical basis for that self-model.
- An agent that skips skill crystallization to install pre-programmed skills will not develop the emergent adaptation properties that make the architecture valuable.

## 16. Limitations and Open Questions

### 16.1 Limitations

**The strongest derivation is L1–L3.** L4 and L5 are architecturally motivated extensions with weaker derivational strength. Readers should calibrate confidence accordingly.

**Non-stationarity is not quantified.** We argue that sufficient environmental change requires L3, but we do not specify what "sufficient" means. A full theory would define a threshold relating environmental change rate to the encoding capacity of L2.

**The architecture is sufficient, not unique.** We argue that this structure is coherent under C1–C3, not that it is the only such structure. Other layered decompositions may also be coherent. Establishing uniqueness would require deeper formal treatment.

**Outside the scope of C1–C3, the framework does not claim to apply.** Static environments, single-task systems, and systems where persistence is not itself valued are not the target.

**Explicit drive injection is an asymmetric bet, not a proof.** We argue that explicit injection dominates emergent drive formation under the instrumental convergence hypothesis. If instrumental convergence is incorrect, our argument weakens though does not collapse. *Fuller formal treatment marked for later development.*

**Anchor formalization is minimal.** The `G(s) → A'(s)` distinction captures the intended difference between anchors and rules but does not constitute a complete theory. A full anchor theory would specify how `A'(s)` is computed, how anchors compose, and the computational trade-offs versus post-hoc filtering.

**Empirical validation remains open.** The framework is intended to be engineering-testable, but the present document does not attempt to maintain implementation progress. Validation of higher-layer behavior remains an open empirical question.

### 16.2 Open questions

**Drive calibration**: Principled basis for initial drive weights and sensitivity curves?

**RPE implementation specifics**: Engineering implementations that preserve the "surprise not magnitude" property?

**Self-model bootstrapping**: The circular dependency between safe early behavior and sufficient history for self-model development?

**Conspecific identification for L5**: Empirical characterization of the agent's social environment?

**Anchor evolution**: Principled process (if any) for modifying constitutional anchors?

**Habit track dynamics**: Decay functions for unused habit tracks?

**Interaction between injected L2 drives and emergent L3 patterns**: Conflict resolution when they diverge?

**The asymmetric bet argument**: Fuller treatment of the conditions under which explicit vs. emergent drive formation is preferable?

**Full formalization of the anchor system**: Moving from the minimal `G(s) → A'(s)` distinction to a complete mathematical treatment?

## 17. Conclusion

EVA is not a proposal for better benchmark agents. It is a proposal for a different kind of problem.

The core claim is simple:

> For a class of agents whose primary property is continuous existence under changing conditions, task-centered design assumptions are structurally insufficient.

Under that reframing, a layered architecture emerges naturally:

- L1 for viability sensing
- L2 for historically compressed pressure and contextual drive
- L3 for within-lifetime adaptation
- with L4 and L5 as later expansions under additional pressures

What makes EVA distinctive is not merely that it uses layers, memory, or biological references. It is the specific combination of:

- **Continuous existence as first-order constraint** (Claim A)
- **Drive as context rather than command**—the most coherent solution for the distributed, non-command coordination EVA requires
- **Anchors as generative structural restriction**—not post-hoc filtering
- **Action selection as a peer circuit**—not a reasoning sub-module
- **Explicit drive injection**—the auditable answer to instrumental convergence

These are not stylistic preferences. Each is an architectural commitment with specific consequences.

The immediate future of EVA is dual:
- Sharpen the theory where it is strongest (L1→L2→L3, anchor formalization)
- Continue empirical and engineering evaluation in the public companion implementation project

The deeper wager is that "agent" has been framed too narrowly for too long. EVA is an attempt to widen that frame—not by adding capability to task-centered agents, but by proposing that for a meaningful class of agents, task-centered framing was never the right starting point.

We do not claim this is the final or the only answer. We claim it is a different answer, and that the difference matters.

## Appendix C: Terminology

### C.1 Core framing terms

| Term | Meaning in EVA |
|---|---|
| Continuous existence | The first-order property EVA is organized around: preserving continuity-relevant viability over time |
| Continuity | Ongoing persistence as the same agent across time, not mere uptime or process duration |
| Existence-centered agent | An agent architecturally structured around continuity rather than task completion; the subject of Claim A |
| Task-centered agent | An agent whose architecture is organized around completing assigned tasks; continuity is secondary or optional |
| Long-running agent | An agent that operates over extended time; this alone does **not** make it existence-centered |
| Persistent agent | An agent with memory or state carried across sessions; persistence is necessary for EVA-like agents but not sufficient |
| Self-maintaining system | A weaker, architecture-relevant notion of a system that preserves its own continuity conditions without implying biological life |

### C.2 Motivation and constraint terms

| Term | Meaning in EVA |
|---|---|
| Drive | An internal state with continuously varying intensity that biases processing; not a goal and not a rule |
| Goal | A candidate objective or target state inside deliberation; goals are shaped by drive state but are not equivalent to drives |
| Drive broadcast | Continuous transmission of current drive intensities into L3; functions as context, not instruction |
| Anchor | A structural constraint on the space of generable actions; pre-generative rather than post-hoc |
| Anchor-based constraint | `G(s) → A'(s) ⊆ A(s)`: candidate generation already occurs inside a restricted domain |
| Rule-based constraint | `G(s) → A(s)`, followed by later acceptance or rejection; the prohibited option still appears in candidate space |
| Constitutional anchor | A designer-injected anchor at the constitutional level; not available for ordinary runtime override |

### C.3 Action and learning terms

| Term | Meaning in EVA |
|---|---|
| Peer circuit | An architectural component operating in parallel with another rather than being subordinated to it; in EVA, the basal ganglia analog is a peer to reasoning, not its child |
| Default inhibition | The resting action state: actions are suppressed unless actively released |
| Mediated release | The architectural requirement that candidate actions pass through selection before execution |
| RPE | Reward prediction error: actual − expected; encodes surprise rather than raw magnitude |
| Habit track | A basal-ganglia-side action pathway for familiar, reinforced sequences that need less full deliberation |
| Skill crystallization | Emergent consolidation of repeatedly successful action into a more direct and reusable pathway |
| Slow variable | A parameter that changes gradually through accumulated experience rather than immediate reaction |

### C.4 Information-carriage terms

| Term | Meaning in EVA |
|---|---|
| Non-design-time information carriage | Information retained and acted on beyond what was encoded at initialization; the essential function of L3 |
| Level 1 information (individual memory) | Information acquired in the agent's own operational lifetime |
| Level 2 information (social learning) | Information acquired from other agents through observation or interaction |
| Level 3 information (cumulative culture) | Information preserved in symbolic systems and social accumulation beyond any single agent |

### C.5 Distinctions that should not be collapsed

- **Continuous existence** is not the same as unconstrained self-preservation.
- **Drive** is not the same as **goal**, and neither is the same as a **rule**.
- **Anchor** is not just a stronger post-hoc filter; it changes candidate-space formation.
- **Long-running** or **persistent** is not the same as **existence-centered**.
- **Stronger LLM access** is not the same as the agent having its own continuity-bearing memory.
